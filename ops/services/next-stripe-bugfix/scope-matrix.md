# next-stripe-bugfix Scope Matrix

更新日: 2026-03-12

## 判定の基本
- まず `原因` を見る
- 次に `対象フロー` を見る
- 次に `対象エンドポイント` を見る
- 断定できなければ `undecidable` で止める

## same_cause_likely
- Webhook署名検証エラーが原因で、複数イベントが同じ受信口で失敗している
- mode の取り違えが原因で、本番だけ複数画面に反映漏れが出ている
- 同じ API 例外処理漏れが原因で、同一フロー内の表示崩れと保存失敗が連動している

## different_cause_likely
- Webhook署名エラー修正後に、別エンドポイントのDB更新不具合が見つかった
- Checkout 成功後の反映漏れとは別に、Billing Portal 側で契約解除フローが壊れている
- 同じ画面に見えても、片方は設定値不足、もう片方は別APIの認証失敗

## undecidable
- 症状は似ているが、ログがなく原因がまだ分からない
- フローが複数またがっており、どこが主因か未確定
- 依頼者の説明だけでは同一原因か別原因かを切れない

## 文面ルール
- same のとき: `同一原因の範囲として対応します`
- different のとき: `原因が別のため、追加対応になります`
- undecidable のとき: `この時点では追加対応の要否は未確定です`
