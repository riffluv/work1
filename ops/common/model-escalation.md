# モデル使い分け

更新日: 2026-03-12

## 基本方針
- 日常運用は `表 + docs + Codex + 人手確認` を優先する
- 高価モデルは常時監視に使わない
- 迷いが大きい場面だけ二重確認する
- Codex は主担当かつ最終統合担当とし、外部AIの案も妥当なら採用する
- 外部AIの提案は、出所ではなく具体性・整合性・再現性で採否を決める

## 役割
- Codex
  - 主担当
  - 実装、差分作成、docs更新、返信文の初版
  - 外部AIの提案を採否判断し、正本へ反映する
- Claude
  - 日本語の自然さ、説明の滑らかさ、違和感の除去
- Gemini
  - 別案の洗い出し、安価な再確認、見落とし検出
- ChatGPT Pro
  - ワークスペース監査、失敗モード分析、skill/運用設計レビュー

## エスカレーション条件
- 価格判断に直結する
- 規約リスクがある
- スコープが `undecidable`
- 納品文の見え方が気になる
- 将来サービス候補かどうか判断したい
