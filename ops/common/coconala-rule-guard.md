# ココナラ運用ガード

更新日: 2026-03-12

## 目的
- 返信文や提案文の前に、規約事故と運用事故を止める。
- skill に暗黙で覚えさせず、毎回同じ境界で判断する。

## 絶対に崩さない境界
- 外部連絡へ誘導しない
- 外部共有へ誘導しない
- 外部決済へ誘導しない
- `.env` の値、APIキー値、Webhook秘密値を受け取らない
- 依頼者リポジトリへ直接 push しない
- 本番デプロイを請けない
- 正式納品前に、未提供の内容を完了扱いしない

## 外部共有の扱い
- ファイル共有はココナラの添付機能を優先する
- GitHub、Google Drive、Dropbox、Notion、外部URL共有をこちらから案内しない
- 容量や形式に問題がある場合も、まずはトークルーム内で分割添付や関連ファイルの絞り込みを案内する

## 事前相談の扱い
- 未購入段階では、切り分けに必要な最小限の質問に留める
- 診断レポート相当の詳細な価値提供を先出ししない
- 料金や追加対応は、根拠が弱い段階で断定しない

## 見積りprequoteの無償/有償ライン
### 無償で行ってよいこと
- 相手文から事実を抽出し、サービス適合を判断する
- 必須3点（期待挙動 / 実際の挙動 / 発生環境）を質問する
- 回答を受けて `15,000円 / 保留 / 断る` のどれかに振り分ける
- 「1件扱いになりそうか」「この時点では情報不足か」を短文で伝える
- 金額と進め方を提示する
- ログやスクショを見て、方向性を1行だけ伝える

### 有償ラインに入ること
- ログやコードを読み込んで原因候補を複数列挙する
- 原因候補ごとの検証方法や修正方針を説明する
- 修正ファイルやpatchを作成する
- 再現検証を行う
- 診断レポート相当の情報を文面で返す

### 判断に迷うとき
- 迷ったら `原因候補の詳細は購入後にお伝えします` で止める
- `15,000円 / 保留 / 断る` の判定が割れたら、追加質問は1回までに留める
- 2回目も判定不能なら、辞退または人手確認へ倒す

## 送信禁止パターン
- `無料で対応します`
- `今回だけ特例でやります`
- `GitHubに招待してください`
- `Driveに置いてください`
- `値ごと.envを送ってください`
- `こちらで本番反映します`
- `まず再決済してください`
- `Webhookを本番で再送してください`

## 安全な言い換え
- `同一原因の範囲として対応します`
- `この時点では追加対応の要否は未確定です`
- `キー名のみで大丈夫です`
- `トークルーム添付で共有をお願いします`
- `最終確認は依頼者様環境でお願いします`

## 人手確認が必須の場面
- 価格に影響する
- 規約リスクがある
- 主力サービスへの紐付け可否が揺れる
- 正式納品ボタンを押す直前
- 同一原因か別原因かを断定しきれない
