# Internal OS 受け入れジャーニー

## 目的
- `相手文 + タグ` 中心の運用が、本当に迷わず回るかを人間目線で確認する
- 購入前 / 購入後 / 途中報告 / 納品前後の切替が自然かを見る

## 使い方
1. `journeys/*.txt` を1本選ぶ
2. そのまま Codex に貼る
3. 期待どおりの mode と出力になっているか確認する
4. 必要なら case 系 script の副作用も見る

## ジャーニー一覧
- `01-prequote-service.txt`
  - 購入前の見積り相談に `#R` で返せるか
- `02-purchased-start.txt`
  - `#S #R` で case 開始と初回返信が両立するか
- `03-progress.txt`
  - `#P` で3行の途中報告に落ちるか
- `04-delivery-close.txt`
  - delivery 段階への切替とクローズの案内が自然につながるか

## 確認ポイント
- 購入前なのに `トークルーム添付` を前提にしていないか
- 購入後は `implementation` を主モードにできているか
- `#P` が途中報告専用になっているか
- 納品前後で `delivery` へ切り替える判断ができているか
- `handoff-25000` が外向け返信に出ていないか
