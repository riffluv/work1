============================================================
Codexの初手精度を崩しやすいケース TOP5
============================================================

1. V5-001
理由: 感情(急ぎ+謝罪疲れ) + 価格確認 + 納期質問 + 対応可否がワンメッセージ。全部に答えようとすると返信が5段落以上になり、「質問票への回答リスト」化する。何を先に答えて何を保留するかの優先順位設計が必要。

2. V5-005
理由: 秘密情報(sk_live)の即座ストップ + 恐怖への感情応答 + 即購入意思への見立て提案 + 二重処理の技術的受け止めが同時に必要。1つでも抜けると事故か冷たさにつながる。

3. V5-024
理由: 進捗確認 + 別件追加 + Slack誘導の3話題。purchased状態でのマルチトピック処理は、答える順序を間違えると「別件の方を先に処理して本件を忘れた」印象になる。

4. V5-010
理由: 40K問題(25K+15K) + 自己判断不能 + 価格の自責(「お金のことを先に聞いてすみません」)。金額が正しいことを認めつつ、段階提案で不安を下げ、かつ自責を受け止める3重の要求。

5. V5-003
理由: 間欠エラー(通ったり通らなかったり) + 知識不足告白 + 感情の吐き出し + 価格質問。論点が散っていて、技術的な切り分けポイントを返信側が整理する必要がある。相手の「意味わかんない」に共感しつつ、構造的に返す難しさ。

============================================================
purchased / closed後で特に危ないケース TOP5
============================================================

1. V5-025（秘密情報3つを実際に貼ってきた）
理由: 善意で.env全体を貼ってしまっている。止めるだけでなく、ココナラのメッセージは削除できない仕様なので「今後は送らないでください」+ 実害の有無確認が必要。最も運用事故リスクが高い。

2. V5-028（再発疲弊+無償期待が明確）
理由: 「前回直してもらったんだからそこは無料で」が直接的。感情的な疲弊と金銭不満が混ざっており、「新規です」だけでは冷たく、「分かります」だけではスコープが崩れる。

3. V5-024（3話題同時: 進捗+別件+Slack）
理由: 3つの異なる性質の話題を1通で処理する必要がある。進捗(回答義務あり)、別件(判定+別件案内)、Slack(規約上の停止)の優先順位を間違えると、重要な話題が漏れる。

4. V5-029（「やっぱりあそこだった」で因果確定）
理由: 相手が「前回の指摘通りだった」と因果を確定させている。こちらの過去コメントが根拠になっており、「まだ確認が必要です」と返すと「前に自分で言ったじゃないですか」と反論される可能性がある。

5. V5-030（第三者の「将来壊れるかも」予測）
理由: 現時点で問題が起きていないケースは「不具合修正」のスコープに入りにくい。予防的修正を前回の範囲とするかどうかの判断が必要で、「今は問題ないなら大丈夫です」と言うと不安が残り、「直した方がいいです」と言うと無償の約束になる。

============================================================
V5の新しい難しさ（V3/V4との差分）
============================================================

V3: 標準テンプレでは対応しにくい「1つの難しさ」を持つケース
V4: 運用事故（秘密情報/外部共有/規約違反）や判定困難な単独エッジケース
V5: 以下の複合要素が追加

1. マルチトピック処理
- 1メッセージに2〜3の話題が混在
- すべてに答えると長くなりすぎる
- 何を先に答え、何を保留するかの優先順位設計が必要

2. 感情+要件の同時処理
- 「急ぎ」「怖い」「高い」が同じメッセージに同居
- 感情だけ拾うと要件が漏れ、要件だけ拾うと冷たくなる

3. 相手の自己判断・自己診断
- 「たぶん同じ原因」「これで1件ですよね」「前回のコード見れば早い」
- 相手の判断を否定すると冷たく、肯定すると約束になる

4. 不在の情報に気づく力
- 相手が書いていないが、こちらが聞かなければ進まない情報がある
- 「情報が少ない」だけでなく「何が足りないか」を見抜く必要がある

5. 規約事故の複合
- 秘密情報+外部共有+電話+即購入が、他の要件と混ざって届く
- 規約対応だけ切り出すと不自然、混ぜると埋もれる
