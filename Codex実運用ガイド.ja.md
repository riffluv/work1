# Codex実運用ガイド

このファイルは、実際のココナラ運用で「今どう投げればいいか」を迷わないためのガイドです。
起動方法の正本は [次セッション用_起動プロンプト.txt](/home/hr-hm/Project/work/次セッション用_起動プロンプト.txt) を使います。
このガイドは、起動後の実務の回し方だけに絞っています。

## まず覚えること
- 相手文だけ貼ると、基本は分析で返る
- 送信用がほしい時は `#R`
- 購入後の途中報告は `#P`
- 下書きで見たい時は `#D`
- 分析だけでいい時は `#A`
- 購入前か購入後か、どこで聞かれたかを一緒に書くと事故が減る

## 起動
- 新セッションを開いたら、[次セッション用_起動プロンプト.txt](/home/hr-hm/Project/work/次セッション用_起動プロンプト.txt) をそのまま使う
- 起動後は、このガイドを見ながら投げる

## 基本の投げ方
一番大事なのは、次の3点です。

1. どこで聞かれたか
2. 今どうしてほしいか
3. 相手文そのもの

おすすめの型:

```text
サービス経由の見積り相談です
#R
相手文: ...
```

```text
プロフィール経由の見積り相談です
#A
相手文: ...
```

```text
購入後のトークルームです
#R 受領返信
相手文: ...
```

```text
購入後のトークルームです
#P
相手文: ...
```

## 場の書き方
迷ったらこの4つで十分です。

- `サービス経由の見積り相談です`
- `プロフィール経由の見積り相談です`
- `メッセージでの相談です`
- `購入後のトークルームです`

厳密な英語タグは不要です。
日本語で場が分かれば十分です。

## 返信文を作る時のコツ
- 相手文だけだと分析で止まりやすい
- 送信用なら `#R` を必ず付ける
- 購入後の途中報告なら `#P`
- どこで聞かれたかを書いておく
- 相手の文をそのまま貼る
- 必要なら一言だけ補足する

補足例:

```text
サービス経由の見積り相談です
#R
相手文: ...
補足: 料金内に収まるかを気にしていそうです
```

```text
購入後のトークルームです
#P
補足: 非エンジニア向けに重すぎない文で
相手文: ...
```

## タグの意味
- `#R`: 送信用返信文
- `#A`: 分析のみ
- `#D`: 下書き
- `#P`: 購入後トークルームの途中報告
- `#S`: 購入確定後の案件記録開始
- `#M`: 重要判断の中間保存
- `#C`: クローズ保存

注意:
- `#M` はお客さん向け返信ではない
- `#P` は途中報告専用
- `#P` は完了報告や納品案内には使わない

## よく使う実務パターン

一覧:
- 見積り相談: `#R` or `#A`
- 購入直後: `#R 受領返信`
- 途中報告: `#P`
- 原因候補が見えたら: `#R 方向性返信`
- 納品前: `#R 納品前確認`
- 納品後: `#R 正式納品案内`

### 1. 見積り相談の返信を作る
```text
サービス経由の見積り相談です
#R
相手文: ...
```

これで Codex 側は、見積り相談として判定しやすくなります。
価格・スコープ・質問数も崩れにくくなります。

### 2. まず分析だけ見たい
```text
サービス経由の見積り相談です
#A
相手文: ...
```

送信用文面はいらない時に使います。

### 3. 購入直後の受領返信
```text
購入後のトークルームです
#R 受領返信
相手文: ...
```

資料を受け取った直後や、最初の一通に使います。

### 4. 購入後の途中報告
```text
購入後のトークルームです
#P
相手文: ...
```

`#P` は進捗共有専用です。
骨格は次の3行です。
- `進捗の共有です。`
- `どこまでは正常で、現在どこを見ているか`
- `次にやること` と `次回連絡目安` または `必要情報`

### 5. 原因候補が見えた時の返信
```text
購入後のトークルームです
#R 方向性返信
相手文: ...
```

### 6. 納品前の確認
```text
購入後のトークルームです
#R 納品前確認
今回の変更点: ...
確認してほしいこと: ...
```

### 7. 正式納品案内
```text
#R 正式納品案内
今回の修正内容: ...
確認手順: ...
```

## 迷ったらこの順で考える
1. 購入前か購入後か
2. 返信文がほしいか、分析だけでいいか
3. 受領直後か、途中報告か、原因説明か、納品前後か

これでだいたい決まります。

## 保存系ショートカット
- `#S`: 購入確定後に案件メモを開始
- `#M`: 重要判断が出た時だけ使う
- `#C`: クローズ時に使う

`#M` を使う目安:
- 同一原因 / 別原因の判定
- 追加料金の切り分け
- 感情注意が必要と分かった時
- 納品方針が固まった時

## 別AIに見てもらいたい時
そのままこう言えばよいです。

- `claudeに聞いてみて`
- `geminiに聞いてみて`
- `chatgptに聞いてみて`
- `両方に聞いてみて`

対象を書かない場合は、直前の Codex 出力が監査対象になります。
条件を付けたい時は、そのまま追記します。

例:
```text
claudeに聞いてみて。直前の返信文が硬くないか見て
```

```text
chatgptに聞いてみて。購入率を落とす言い回しがないか見て
```

## よくある事故
- `#R` を付けずに送って、分析だけ返ってくる
- 購入前なのに `トークルーム` 前提の文を作らせる
- `#M` をお客さん向け返信だと勘違いする
- 購入後の途中報告なのに `#R 正式納品案内` を使う
- どこで聞かれたかを書かずに、場を取り違える
- 相手文を貼り忘れて、場とタグだけで投げてしまう
