# ココナラ初月運用 1ページ要約（Next.js/Stripe不具合診断）

更新日: 2026-02-13
対象カテゴリ: IT相談・システム開発 > プログラミング・ソフト開発 > バグ修正・コード改修代行

## 1. 結論（今の運用方針）
- 現行価格は維持: 基本15,000円 / 追加修正1件15,000円 / 追加調査30分5,000円
- 公開後2週間は「価格変更」より「返信速度・見積精度・FAQ改善」を優先
- 初回返信は早く、本文は短く、次アクションを明確化
- スコープ外は曖昧に受けず、追加対応として明確に切り分け

## 2. 採用するポイント
- 初回返信を速くする（受領 + 次の報告時刻を先に伝える）
- FAQで先回りする（範囲、追加料金、納品後対応、最終確認フロー）
- 追加料金は「共感 -> 範囲外説明 -> 金額提示 -> 承認後着手」の順で案内
- 進捗連絡は最低1日1回、遅延見込みは早めに連絡

## 3. 採用しないポイント
- 新規だから3,000円開始を絶対視する方針
- 「必ず」「確実」「最短」など断定表現
- 過度な返金保証の常設（条件が曖昧だとリスク増）
- 不確かな外部ノウハウをそのまま運用ルール化

## 4. 公開前チェック（必須）
- タイトル/キャッチコピーに主要語を含む
- サービス内容に「不具合1件の定義」を明記
- 標準範囲外（通話、直接push、本番デプロイ、新機能）を明記
- 最終確認フロー（依頼者環境で実機確認、結果ログで追加調整）を明記
- FAQに追加修正・納品後追加対応・秘密情報方針を記載
- 見積受付はON、購入前見積必須はOFF

## 5. 初月KPI（14日で判定）
- 初回返信: 24時間以内100%（目標は通常24h以内）
- 見積相談数: 週次で計測
- 見積 -> 購入転換率: 週次で計測
- 返信遅延・認識齟齬の発生件数: 週次で計測

## 6. 価格見直しトリガー（2週間後）
- 見積相談が少ない: タイトル/キャッチ/FAQを先に調整し、それでも弱ければ価格調整を検討
- 見積相談は来るが購入率が低い: 見積文面と範囲説明を見直す
- 購入率が十分で対応が逼迫: 価格維持または段階値上げを検討

## 7. 返信文の最短ルール
- 相手文の貼り付けのみ: まず分析のみ
- 「返信文作って」「返信書いて」指定時のみ送信用文面を作成
- 送信用文面は `runtime/replies/latest.txt` に保存してコピペロスを削減
