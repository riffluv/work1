# ココナラ仕様照合メモ（2026-05-01）

## 目的

`#RE` / `#R` の phase 表現が、ココナラ公式仕様と Workspace の記憶領域にずれていないかを確認した。

## 確認した公式仕様

- 見積り相談は、購入前に相談し、出品者が見積り提案を送り、buyer がその提案を購入する流れ。
  - 公式: https://coconala-support.zendesk.com/hc/ja/articles/900004238843-%E8%A6%8B%E7%A9%8D%E3%82%8A%E6%A9%9F%E8%83%BD%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6-%E5%87%BA%E5%93%81%E8%80%85%E5%90%91%E3%81%91
- 見積り提案後は、購入期限までメッセージ機能でやり取りし、buyer の入金完了後にトークルームがオープンして取引開始。
  - 公式: https://coconala-support.zendesk.com/hc/ja/articles/360004844794-%E8%A6%8B%E7%A9%8D%E3%82%8A%E7%9B%B8%E8%AB%87%E3%81%AE%E6%8F%90%E6%A1%88%E3%82%84%E8%BE%9E%E9%80%80%E3%81%AE%E6%B5%81%E3%82%8C-%E3%82%B5%E3%83%BC%E3%83%93%E3%82%B9%E7%B5%8C%E7%94%B1-%E3%83%97%E3%83%AD%E3%83%95%E3%82%A3%E3%83%BC%E3%83%AB%E7%B5%8C%E7%94%B1
- メッセージ機能は購入前相談・クローズ後連絡に使えるが、正式な依頼はサービス購入後のトークルームで行う。
  - 公式: https://coconala-support.zendesk.com/hc/ja/articles/218721057-%E3%83%A1%E3%83%83%E3%82%BB%E3%83%BC%E3%82%B8%E6%A9%9F%E8%83%BD%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6
- トークルームは購入後から取引完了までのやり取りの場。
  - 公式: https://coconala-support.zendesk.com/hc/ja/articles/218177948-%E3%83%88%E3%83%BC%E3%82%AF%E3%83%AB%E3%83%BC%E3%83%A0%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6
- クローズ後はトークルームでメッセージ、ファイル投稿、おひねり等の追加購入ができない。連絡はメッセージ機能へ戻る。
  - 公式: https://coconala-support.zendesk.com/hc/ja/articles/218723147-%E3%83%88%E3%83%BC%E3%82%AF%E3%83%AB%E3%83%BC%E3%83%A0%E3%81%AE%E3%82%AF%E3%83%AD%E3%83%BC%E3%82%BA%E5%BE%8C%E3%81%AB%E8%BF%BD%E5%8A%A0%E3%81%AE%E8%B3%AA%E5%95%8F%E3%82%84%E4%BE%9D%E9%A0%BC%E3%82%92%E3%81%97%E3%81%9F%E3%81%84
- メッセージ機能・トークルームともにファイル添付は可能。容量はメッセージ 100MB、トークルーム 200MB。
  - 公式: https://coconala-support.zendesk.com/hc/ja/articles/218180658-%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E6%B7%BB%E4%BB%98%E6%A9%9F%E8%83%BD%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6

## 照合結果

### 一致している

- `prequote`: 見積り提案前の購入前相談として扱う。
- `quote_sent`: 見積り提案済み・支払い前として扱う。
- `purchased`: 入金完了後にトークルームが開いた取引中として扱う。
- `delivered`: 正式な納品後・クローズ前として扱う。
- `closed`: クローズ済み。旧トークルームでの作業継続、おひねり、ファイル投稿を前提にしない。
- `closed` 後は、メッセージ上で概要・前回修正との関係を確認し、実作業が必要なら見積り提案または新規依頼へ戻す。

### 修正した注意点

古い Gold 33 に、`quote_sent` では `トークルーム内` と書かない、という強めの表現が残っていた。

公式仕様上は、quote_sent ではまだトークルームは開いていないが、入金完了後にはトークルームが開く。
そのため、次のように分ける。

- NG: quote_sent で、現在すでにトークルームで作業中のように書く。
- OK: `お支払い完了後にトークルームで共有してください` のように、未来条件つきの購入後手順として書く。

あわせて、`prequote` ではまだ見積り提案済みではないため、`見積り提案の内容で問題なければ` を使わないことを `platform-contract` / `acceptance-gate` / bugfix skill に明記した。

## #RE への適用

- fixture の `state` だけでなく、buyer 文にも `見積り提案ありがとうございます` / `購入後です` / `クローズ後です` など phase が見える語を置く。
- `quote_sent` では、支払い前作業開始を止める。ただし、未来手順として `お支払い完了後にトークルームで共有` は使ってよい。
- `prequote` では、まだ提案済みではないため、購入導線を出す場合も `この内容で進める場合は、そのままご購入ください` など公開サービス購入に寄せる。見積り提案済みの表現へ進めない。

## 判定

現行の中心記憶領域は、公式仕様と大きく一致している。
今回の問題は「ココナラ仕様を知らない」より、古い Gold 表現が強すぎて quote_sent の未来手順まで縛りかねない点だった。
