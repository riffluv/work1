# #R Smoke Style Anchors

作成日: 2026-04-29

## 位置づけ

- これは Gold Reply ではなく、`#RE` を本丸 `#R` に近い writer candidate へ戻すための一時 style anchor。
- 目的はテンプレ化ではなく、Codex の writer 判断で出せている「職人寄りの実務チャット感」を、`#RE` の監査候補にも反映すること。
- service / scope / phase / secret / public-private / payment route は変更しない。

## 観察された良い型

- 相手の語を1点拾ってから、本題へ入る。
- 主質問に先に答える。
- サービスページ調の説明をそのまま貼らず、相手文に接続する。
- 事実と次アクションを近い段落に置く。
- `確認します` / `お返しします` だけの処理文で終わらせない。
- closed 後は、送ってよいかへの答えを先に置き、その後で実作業境界を出す。

## Anchor 1: 金額 + 納期

相手文:

> Stripe決済後に注文が作られないです。金額だけと、だいたい何日かかるかだけ先に教えてください。

返信:

> ご相談ありがとうございます。
>
> 金額は15,000円です。
> 今回のような「Stripe決済後に注文が作られない」症状であれば、不具合1件として原因確認から修正済みファイルの返却まで対応できます。
>
> 日数はサービス上3日が目安です。
> ただ、コードの状態やエラー内容によって前後するため、ご購入後に内容を確認できた時点で、より正確な見通しをお返しします。
>
> この内容で進める場合は、そのままご購入ください。

## Anchor 2: 原因不明でも相談できるか

相手文:

> Stripe決済後に注文が作られないのですが、原因がまったく分かっていません。この状態でも相談して大丈夫ですか？15,000円で見てもらえますか？

返信:

> ご相談ありがとうございます。
>
> 原因が分からない状態でも大丈夫です。Stripe決済後に注文が作られない不具合として、15,000円で原因確認から修正済みファイルの返却まで進めます。
>
> ご購入後は、コードとエラー内容を見ながら、決済後の処理がどこで止まっているかを確認します。必要な情報を確認できた時点で、一次結果は48時間以内を目安にお返しします。
>
> この内容で進める場合は、そのままご購入ください。

## Anchor 3: 受領確認 + 次の流れ

相手文:

> 昨日、StripeのEvent IDが分かるログとスクショを送りました。届いていますか？この後の流れと、追加で用意した方がいいものがあるかだけ教えてください。

返信:

> ログとスクショありがとうございます。
> StripeのEvent IDが分かるログとスクショは届いています。
>
> この後はこちらで内容を見ながら、決済後に注文作成まで進んでいるかを確認します。
> いまの時点で追加で準備いただくものはありません。足りないものが出た場合だけ、こちらから具体的にお伝えします。
>
> 本日22:30までに、分かっている点を一度お返しします。

## Anchor 4: 短い進捗確認

相手文:

> 昨日購入してログも送りました。今どこまで見ていますか？長い説明はいりません。現時点で分かっていることだけ短く教えてください。

返信:

> ありがとうございます。ログは受け取っています。
> 先に現状だけお伝えします。
>
> 今は、決済後の反映処理がどこで止まっているかを追っています。
> まだ原因の断定まではしていませんが、注文作成に入る前後の処理を中心に見ています。
>
> 本日22:30までに、分かっている点だけ短くお返しします。

## Anchor 5: closed 後の関係確認

相手文:

> クローズ後ですみません。似たStripeエラーが出ています。ログとスクショをこのメッセージで送れば、前回の修正と関係あるかだけ見てもらえますか？実際に直す必要がありそうなら新規で相談します。

返信:

> ご連絡ありがとうございます。
> ログとスクショは、このメッセージで送っていただいて大丈夫です。
>
> 届いた内容をもとに、前回の修正と関係がありそうかだけ先に見立てます。
> トークルームはクローズ済みのため、この場で修正作業や修正済みファイルの返却までは進められません。
>
> 実作業が必要そうな場合は、見積り提案または新規依頼として、対応方法を先にご相談します。

## #RE へ戻す時の注意

- 上記を全文テンプレとして固定しない。
- `今回のような` は、相手文の症状が具体的な時だけ使う。
- 報告時刻は送信時刻に合わせて差し替える。
- `#RE` では writer candidate の質を見るために使い、renderer baseline へ固定文として埋め込まない。
