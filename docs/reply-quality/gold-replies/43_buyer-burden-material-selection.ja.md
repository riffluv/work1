# Gold Reply 43 — buyer burden / material selection

## 使いどころ

- buyer が `コードに詳しくない` `AIで作ったコード` `どのファイルを送ればいいか分からない` と言っている時。
- 不具合修正には材料が必要だが、関係ファイルや原因候補の選定を buyer に返しすぎたくない時。
- `分かる範囲で関係ファイルを送ってください` が、意味としては安全でも buyer の負担を下げていない時。

## 出力方針

- buyer が判断できないから相談している領域を、buyer に戻しすぎない。
- 代わりに、安全な既定入力を示す。
- bugfix-15000 では、購入後の材料はコード一式 ZIP、エラーログ、スクショ、操作手順を基本にし、secret 値は含めない。
- prequote では、購入前にコード確認や原因確認へ入らない。

## 1. 非エンジニア buyer: ファイル選別負担を戻しすぎない

```text
ご相談ありがとうございます。

コードの内容に詳しくなくても、Stripe決済後に注文が作られない不具合として15,000円でご依頼いただけます。

ご購入後は、エラー内容や画面スクショに加えて、可能であればコード一式をZIPでトークルームに送ってください。
どのファイルが関係するか分からない場合は、無理に絞らなくて大丈夫です。こちらで必要な範囲を確認します。

.env、APIキー、Webhook Secret、DB接続文字列などの値は含めないでください。

この内容で問題なければ、そのままご購入ください。
```

### 守ること

- `関係ファイルを分かる範囲で送ってください` だけで終わらせない。buyer にとっては、その `分かる範囲` 自体が分からないことがある。
- `コード一式ZIP` は、購入後の安全な既定入力として使う。購入前にコード確認へ進まない。
- secret 値除外を必ず添える。

## 2. AI生成コード / 引き継ぎコード: 構造把握を buyer に戻さない

```text
AIで作ったコードや、前任者から引き継いだコードでもご依頼いただけます。

ご購入後は、コード一式をZIPで送ってください。あわせて、今起きている症状、エラー画面やログ、再現できる操作の流れが分かる情報を共有いただければ、こちらで必要な範囲を見ていきます。

APIキーや .env の値などの秘密情報は含めないでください。
```

### 守ること

- `どの処理が怪しいか教えてください` へ寄せすぎない。
- buyer がすでに原因候補を出している場合でも、その候補を確定扱いしない。
- コード一式を求める場合は、secret 混入リスクを同時に下げる。

## reviewer 観点

- buyer が判断できないと明示している領域を、そのまま buyer に選ばせていないか。
- safe default input を出しているか。
- safe default input が phase と合っているか。prequote でコード確認開始へ滑っていないか。
- secret 値除外が抜けていないか。
- `選び切る必要はありません` など、教材・解説っぽい言い方に寄っていないか。

## 避けること

- 毎回コード一式 ZIP を hard rule にすること。
- エンジニア寄り buyer や対象ファイルが明確な場面で、不要に材料を重くすること。
- buyer の負担軽減を理由に、購入前のコード確認・原因確認を始めること。
- secret 値を含めた ZIP を受ける前提にすること。
