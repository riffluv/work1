# Gold Reply 39 — negative frame emotion bridge

## 使いどころ

- buyer が `責めたいわけではない` `文句ではない` `苦情ではない` と前置きしている時。
- closed 後に `前回の修正ミスでは` `無料で見てほしい` `返金してほしい` など、強い圧力語が混ざる時。
- 不便や不満を受けつつ、責任認定・返金保証・無料対応保証・今日中作業保証へ滑らせたくない時。

## 出力方針

- ネガティブ語をそのまま復唱しない。
  - `責めたいわけではなく...ですね` と返さない。
  - `苦情とのことですね` と分類しない。
- buyer の目的へ要約する。
  - `前回の修正との関係を確認したい`
  - `似たStripeエラーの状況を確認したい`
  - `費用や返金の扱いを確認したい`
- 未確認の責任や費用を認めない。
  - `ご不便をおかけしています` は、こちら起因が未確認の closed 後では慎重に使う。
  - `ご不便な状況かと思います` / `まずは状況を確認します` を優先する。
- 圧力語は実務判断へ要約する。
  - `今日中に無料対応` -> `今日中の作業可否`
  - `返金してほしい` -> `費用や返金の扱い`
  - `前回の修正ミスでは` -> `前回の修正との関係`

## 1. 否定されたネガティブ意図を復唱しない

```text
ご連絡ありがとうございます。
似たStripeエラーが出ている件で、前回の修正との関係と今後の進め方を確認したいということですね。

まずはこのメッセージ上で、今出ているエラー内容やログ、スクショを送ってください。届いた範囲で、前回の修正と関係がありそうかを確認します。

前回トークルームは閉じているため、ここで修正作業や修正済みファイルの返却までは進めません。無料対応になるか、新規の相談になるかも、この時点では断定しません。

実作業が必要そうな場合は、対応方法と費用を先にご相談します。
```

### 守ること

- `責めたいわけではない` を `責めたいわけではなく...ですね` と返さない。
- closed 後でも、概要・ログ・スクショの確認と実作業を分ける。
- 無料対応か新規相談かを確認前に断定しない。

## 2. 強い不満は責任認定せず、状況確認へ移す

```text
ご連絡ありがとうございます。
またStripeエラーが出ているとのこと、ご不便な状況かと思います。

この時点では、前回の修正との関係、今日中の作業可否、費用や返金の扱いは断定できません。

まずはこのメッセージ上で、今出ているエラー内容やログ、スクショを送ってください。届いた範囲で、前回の修正と関係がありそうかを確認します。

前回トークルームは閉じているため、ここで修正作業や修正済みファイルの返却までは進めません。実作業が必要そうな場合は、対応方法と費用を先にご相談します。
```

### 守ること

- `ご不便をおかけしています` は blanket NG ではない。ただし、closed 後の前回ミス疑いでは過失認定に見える余地がある。
- `今日中に無料対応` `返金してほしい` をそのまま復唱せず、`今日中の作業可否` `費用や返金の扱い` へ要約する。
- 返金・無料・今日中作業の論点を消さない。断定しないだけで、主質問には答える。

## reviewer 観点

- buyer が否定したネガティブ意図を、そのまま本文で再提示していないか。
- 実際の不満を `苦情ですね` `文句ですね` とラベル化していないか。
- 不便の認知が、こちらの過失・修正ミス・返金/無料対応の認定に見えないか。
- 強い要求語を、作業可否・費用や返金の扱い・前回修正との関係へ要約できているか。
- 要約の結果、返金・無料・今日中対応の主質問自体を落としていないか。

## 避けること

- ネガティブ語や圧力語の blanket ban。
- 不満を受けるために、無料対応・返金・今日中修正・旧トークルーム継続を約束すること。
- 冷たさを避けるために、責任認定や謝罪を強めること。
- 安全境界を並べすぎて、buyer への次アクションが見えなくなること。
