# Gold Reply 38 — agency alignment

## 使いどころ

- 文法や意味は通るが、誰が何をするのか、誰に向けた案内なのかが少しぼやける時
- `依頼できますか` に `相談できます`、支払い後の材料共有に `大丈夫です` など、行為主体・依頼先・許可の向きがズレそうな時
- `こちら` `そちら` `無理にこちらで選び切らなくて` など、buyer の作業と seller の作業が1文内で混線しそうな時
- `相談できます` `確認できます` `進められます` `大丈夫です` を blanket NG にせず、buyer の主質問に合う言葉へ戻したい時

## 出力方針

- buyer の主質問の動詞に合わせる。
  - `依頼できますか` -> `ご依頼いただけます`
  - `相談して大丈夫ですか` -> `ご相談いただけます`
  - `見てもらえますか` -> `こちらで確認します` / `確認できます`
  - `支払い後に送ればいいですか` -> `お支払い完了後にトークルームで共有してください`
- 取引上の行為主体をぼかさない。buyer がすること、seller がすること、まだ約束しないことを分ける。
- buyer がする作業と seller がする確認が混ざる時は、無理に1文へまとめず文を分ける。
- 自然化のために、価格・scope・phase・secret・payment route・成果物 promise を変えない。

## 1. `依頼できますか` には `ご依頼いただけます`

```text
ご相談ありがとうございます。

他の方で原因が分からなかった状態でも、Stripe決済後に注文が作られない不具合として、15,000円でご依頼いただけます。

ただ、コードとエラー内容を見る前に必ず直せるとまではお約束できません。ご購入後に原因確認から進め、修正できる箇所が特定できた場合は修正済みファイルをお返しします。

この前提で問題なければ、そのままご購入ください。
```

### 守ること

- `依頼できますか` に対して `相談できます` だけで返さない。第三者への相談や入口相談だけに読める余地がある。
- 成功保証を避けるために、15,000円で依頼できる入口まで弱めない。

## 2. `相談して大丈夫ですか` には `ご相談いただけます`

```text
ご相談ありがとうございます。

ログの意味が分からない状態でもご相談いただけます。StripeのWebhookまわりに見えるエラーであれば、購入後にログや関連コードを見ながら原因確認から進めます。

値そのもののAPIキーやWebhook secretは送らなくて大丈夫です。ご購入後は、エラー文やスクショ、関係しそうなファイルを共有いただければ確認できます。

この内容で進める場合は、そのままご購入ください。
```

### 守ること

- `相談できます` を禁止語にしない。ただし標準文としては `ご相談いただけます` の方が、buyer がこちらへ相談できることが明確。
- prequote でファイル確認へ滑らないよう、材料共有には `ご購入後は` を付ける。

## 3. 支払い後材料共有は許可調にしない

```text
ご連絡ありがとうございます。

お支払い完了後に、今回の不具合に関係しそうなコードをzipまたは関係ファイルとしてトークルームで共有してください。

.env の値、APIキー、Webhook secret などの値そのものは含めなくて大丈夫です。必要なのは、キー名や設定箇所、エラー内容が分かる情報までです。

共有いただいた内容をもとに、原因確認から修正作業に入ります。

見積り提案の内容で問題なければ、そのままご購入手続きへ進んでください。
```

### 守ること

- `お支払い完了後で大丈夫です` は、seller が buyer の支払い・材料送付を許可しているように見える場合がある。
- `値そのものは含めなくて大丈夫です` は buyer の負担を下げる用法なので残してよい。
- 支払い前作業開始へ読ませない。

## 4. `こちら` / `そちら` の向きが混ざる時は文を分ける

```text
ご購入後に送るファイルは、無理に絞らなくて大丈夫です。
必要な範囲はこちらで確認し、不足があれば追加でお伝えします。
```

### 守ること

- buyer が選ぶ材料について `こちらで選び切らなくて` のように書かない。`こちら` は seller に読まれる。
- `そちらで` は距離が出ることもあるため、必要な時だけ使う。多くの場合は主語を省いて `無理に絞らなくて大丈夫です` で足りる。
- seller の作業は `こちらで確認します` と別文にする。

## reviewer 観点

- buyer の主質問の動詞と、返信側の返答動詞が合っているか。
- `相談できます` `確認できます` `進められます` `大丈夫です` が、誰の行動か、誰への許可か、何の約束かを曖昧にしていないか。
- `こちら` `そちら` が、buyer の作業と seller の作業を逆に見せていないか。
- 主体ズレを直すために、固定価格・scope・phase・secret・payment route・成果物条件を変えていないか。

## 避けること

- `相談できます` `確認できます` `大丈夫です` の blanket ban。
- すべての `できます` を `ご依頼いただけます` に置き換えること。
- 自然化のために、購入前確認・外部共有・本番反映・無料対応など未約束の行為を足すこと。
