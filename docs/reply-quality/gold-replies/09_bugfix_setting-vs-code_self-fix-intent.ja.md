# Gold Reply 09: bugfix setting-vs-code self-fix intent

## 使う場面
- buyer が `設定の問題ですか？コードの問題ですか？` と切り分けを求めている
- `設定だけなら自分で直したい` という意向も出している
- 価格と対応可否を同時に聞かれている

## 返信例

```text
ご相談ありがとうございます。
設定の問題かコードの問題かは、今の段階ではまだ切り分けが必要です。ただ、決済できない症状として15,000円で対応できますし、設定だけで直せそうならその形も含めて返せます。
決済できない時に出ている画面か文言を送ってもらえますか。
```

## 良い理由
- `設定かコードか` を今は断定しない
- `自分で直したい` という意向を閉じずに拾う
- 価格だけを先に押しつけず、症状ベースで対応可否を返す
- ask は購入前に受けられる粒度で止める
