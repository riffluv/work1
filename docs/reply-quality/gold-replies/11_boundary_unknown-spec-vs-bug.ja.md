# Gold Reply 11: boundary unknown spec vs bug

## 使う場面
- buyer 自身が `不具合なのか仕様なのか分からない` と言っている
- 前任者や外注先から `仕様です` と言われていて判断しづらい
- 断定ではなく、判断基準を buyer に見える形で返したい

## 返信例

```text
ご相談ありがとうございます。
外注先に「仕様です」と言われていると判断しづらいと思います。Stripe決済後のメールが毎回すぐ届く前提なのか、30分遅れるのが想定外なのかで見方が変わるので、まずは今の動きが想定どおりかを確認するところから入れます。
メールが遅れて届いたことが分かる画面か、発生した日時があれば送ってもらえますか。
```

## 良い理由
- `仕様か不具合か` をその場で断定しない
- buyer が判断できる軸を、会話で分かる言葉にして返せる
- いきなり技術調査へ飛ばず、入口として必要な情報だけを頼める
