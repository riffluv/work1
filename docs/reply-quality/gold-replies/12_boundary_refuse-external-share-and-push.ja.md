# Gold Reply 12: boundary refuse external share and direct push

## 使う場面
- GitHub 招待、外部共有、直接 push などを求められる
- 断るだけで終わらず、どう進めるかを同じ返信で見せたい
- platform / contract 制約を守りつつ講義調を避けたい

## 返信例

```text
ご連絡ありがとうございます。
GitHub招待は今は使わず、このトークルームで進めます。直接 master ブランチに push することもしていません。
必要な修正箇所があれば、この場で確認しながら返します。
```

## 良い理由
- 2つの `できません` を並べても、最後の `こう進めます` で会話を閉じられる
- GitHub / Drive / Zoom / 対面など、他の外部共有断りにも流用しやすい
- 制約説明を講義調にせず、代替の進め方をすぐ出せる
