# Gold Reply 08: bugfix の scope 判定を buyer語で返す

## 使う場面
- bugfix の prequote で `対応できますか` `この料金で見てもらえますか` を聞かれている
- 症状はある程度見えているが、内部の scope 判定語に寄りやすい
- `この不具合として対応できます` のような曖昧参照を避けたい

## 返信例
ご相談ありがとうございます。
本番だけ 404 になっている症状であれば、15,000円で対応できます。

まずは、404 になっている画面か出ている文言を送ってもらえますか。
そこが分かれば、このまま進められるかを返せます。

## 良い理由
- `この不具合` を使わず、buyer に見える症状名で scope を返せる
- `対応できます` を先に置いて、内部の判定ラベルに逃げない
- ask も `画面 / 文言` の粒度に止まり、prequote の範囲を超えない
- prequote で `見ます / 切り分けます / 確認します` のような購入後動詞に滑らず、`返せます` で止められる
