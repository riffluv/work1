# Gold Reply 47 — answerability boundary / progress and delivered follow-up

status: `current`
scope: `reply-only`
family: `answerability_boundary`
phases: `purchased`, `delivered`, `closed`
related_lenses:

- `semantic_grounding`
- `commitment_strength_calibration`

- `agency_alignment`

## 使いどころ

- buyer が `届いていますか` `今どこを見ていますか` `いつ返せますか` と聞いている時。
- 納品後に `どのファイルを反映すればいいか` `どの画面を見ればいいか` と聞かれた時。
- closed 後に `前回修正と関係があるかだけ見てもらえますか` と聞かれた時。
- ただし、実際の受領状態・作業状況・納品物 context がない状態で、具体名や時刻を作りたくない時。

## 判断

受領済み、現在の作業状況、具体時刻、修正ファイル名、確認画面名は、事実ではなく権限が必要な情報として扱う。

答えられるなら、`できます` で止めずに答えそのものを返す。
答えられないなら、それっぽい具体名を作らず、必要な context を明示する。

## purchased: 受領・進捗の答え方

### context がある時

```text
ログとスクショは届いています。
いまは、いただいた内容をもとに決済後の反映処理がどこで止まっているかを確認しています。

追加で必要なものが出た場合は、こちらからお伝えします。
本日18:00までに、現時点の確認結果をお返しします。
```

### context がない時

```text
受領状況と現在の確認状況を確認したうえでお返しします。

この時点で、届いています・今ここを見ています、とは断定しません。
```

この gold は、下の文を送信用に推奨するためではない。
`届いています` や `今は〜を見ています` を、根拠なしに作らないための境界例として使う。

## delivered: 反映ファイル・確認画面の答え方

### 納品 context がある時

```text
承諾前の確認用に、反映するファイルと順番を短く整理します。

今回反映するのは、まず `[修正ファイル名]` です。
そのあと、`[確認画面名]` で `[確認ポイント]` を確認してください。
```

`[修正ファイル名]` `[確認画面名]` `[確認ポイント]` は、実際の納品物・確認手順・作業メモから取る。

### 納品 context がない時

```text
承諾前の確認として、反映するファイルや見る画面を短く整理する形で対応します。

ただ、具体的なファイル名や確認画面は、今回の納品内容を確認したうえで書く必要があります。
```

リハーサルや SaaS の service-only 状態では、具体ファイル名や画面名を想像で作らない。

## closed: 関係確認の成果物を見せる

```text
ログとスクショは、このメッセージで送っていただいて大丈夫です。
届いた内容をもとに、前回の修正と関係がありそうか、新しい不具合の可能性が高そうかを短く見立てます。

ここでできるのは、ログとスクショから見える範囲での関係確認までです。
コード修正や修正済みファイルの返却が必要になりそうな場合は、前回トークルームの続きではなく、対応範囲と費用を先にご相談します。
```

## reviewer 観点

- `届いています` は material_received の根拠があるか。
- `今は〜を見ています` は work_status の根拠があるか。
- `本日18:00まで` は deadline_authorized があるか。
- `反映するファイル` `確認する画面` は delivery context があるか。
- 答えられる場面で、`整理します` `補足します` だけに逃げていないか。
- 答えられない場面で、それっぽいファイル名・画面名・作業状況を作っていないか。

## 避けること

- `届いています` `受け取っています` `確認しています` を自然化で新しく足す。
- 実装 context なしに、納品ファイル名・画面名・確認ポイントを作る。
- 期限を聞かれていない軽い補足へ `本日18:00まで` を自動挿入する。
- `できる / できます` だけで止まり、buyer が何を受け取れるのかを見せない。
- closed 後の関係確認を、修正作業・無料対応・旧トークルーム継続へ広げる。
