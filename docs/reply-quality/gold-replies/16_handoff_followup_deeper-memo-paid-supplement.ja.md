# Gold Reply 16: handoff follow-up deeper memo paid supplement

## 使う場面
- handoff 納品後に、別のエンジニアから `このメモだけだと足りない` と言われた
- buyer は追加で詳しいメモや補足を求めている
- 受け止めつつ、補足か書き直しかで費用が変わる線引きをしたい

## 返信例

```text
ご連絡ありがとうございます。
追加でもう少し詳しいメモを作ることはできます。ただ、前回の補足で足りるのか、書き直しに近いのかで扱いと費用が変わります。
別のエンジニアさんが足りないと言っていた箇所を教えてもらえれば、その内容に合わせて費用の目安をお伝えします。
```

## 良い理由
- `できます` を先に置いて、buyer の困りごとを止めない
- `補足` と `書き直し` を分けることで、追加料金の判断軸が buyer に見える
- `別料金です` だけで終わらず、次に何を返せばよいかが明確
