# Gold Reply 01: bugfix 見積り初回（非エンジニア）

## 使う場面
- 非エンジニア寄りの購入前相談
- 決済・反映漏れ・Webhook 周辺だが、相手は技術語に強くない
- 最初から聞きすぎたくない

## 返信例
ご相談ありがとうございます。
技術的なことはこちらで確認しますね。

スクショだけでも進められます。分かるところだけ、次の3つを教えてください。
1) 本来どうなってほしいか
2) 問題が出ている画面スクショ（成功画面 / エラー画面のどちらか1枚で大丈夫です）
3) 本番とテストどちらで起きているか（不明ならそのままで大丈夫です）

※ .env や API キーの値は送らないでください。キー名だけなら大丈夫です。
確認できしだい、2時間以内（JST）を目標にまず方向性をお返しします。
そのあと、48時間以内を目標に一次結果をご案内します。

## 良い理由
- 最初の依頼が3点に収まっている
- 技術語を減らしつつ、必要十分な情報が取れる
- 次回時刻があり、速さへの不信を減らせる
