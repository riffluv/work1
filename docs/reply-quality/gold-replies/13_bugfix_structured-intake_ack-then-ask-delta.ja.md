# Gold Reply 13: bugfix structured intake ack then ask delta

## 使う場面
- buyer が `1) 現象 2) 環境 3) エラー` のように整理して相談している
- 情報はかなり揃っているが、あと1点だけあれば切り分けに入れる
- 既出情報のオウム返しや、広い ask を避けたい

## 返信例

```text
ご相談ありがとうございます。
Vercel本番で `evt_12345` が 500 になって、決済後も `is_premium` が false のままなら、その反映不具合として対応できます。ここまで整理してもらえているので、まずはイベント処理のどこで止まっているかを切り分けます。
`evt_12345` の前後で出ている文言か、500 になっているログがあれば送ってもらえますか。
```

## 良い理由
- buyer がすでに出した情報を並べ直さず、受領したうえで話を進められる
- ask を `足りない1点` に絞れる
- semi-technical な短文相談でも、重くしすぎずに返せる
