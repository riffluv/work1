# ココナラ勝ち筋メモ

更新日: 2026-03-12

## 目的
- 出品直後に迷わないよう、今の主力サービスの勝ち筋を短く固定する。
- `事故率を上げずに最初の高評価コメントを取る` ことを最優先にする。

## 今の主力戦略
- 広く何でも直す人ではなく、`Next.js / Stripe / API連携の1不具合を、安全に切り分け、原因説明つきで返す人` として見せる。
- 特に `決済は成功しているのに反映されない` `本番だけWebhookが失敗する` `AIや外注から引き継いだコードの原因が分からない` を主訴にする。
- 最初の1件は利益最大化より `高評価コメント` を優先する。

## 取りにいく依頼者
- 非技術者で、AIで作ったコードや引き継ぎコードの不具合に困っている人
- 決済や会員状態など、売上や運用に直結する不具合で止まっている人
- 「技術の深さ」より「説明してほしい・安心したい」を求める人

## 今は避けるもの
- 細かいUI調整が中心の案件
- 継続保守前提の案件
- Stripe 以外の決済案件
- Firebase 実機依存が強く、単価も弱い案件
- 複数原因が絡み、最初から1不具合に収まらない案件

## 価格の考え方
- 公開価格15,000円は維持する
- 安売りは最終手段
- 見積り経由では、オプション前提より `今回の範囲を束ねた一式見積り` を優先する
- 実績目的で拾う案件でも、価格の安心感を壊す公開値下げは避ける

## 納品物で勝つポイント
- 結論サマリーを先に出す
- 原因説明を核にする
- 修正箇所と確認手順を追える形にする
- 非技術者でも `ちゃんと直してくれた` と感じる順番で見せる

## プロフィール導線の使い方
- プロフィールは自己紹介だけでなく、サービス外寄り相談の受け皿でもある
- 主力サービスに近い相談は、見積り時に主力サービスへ紐付けて実績に集約する
- ズレるが技術的に現実的な案件は、`service_mismatch_but_feasible` として人手判断へ上げる
