# ココナラ 見積り設定UIチートシート（3行版）

更新日: 2026-02-16

## まずこの3行だけ覚える
- `購入者から「見積り相談」を受け付ける = サービスページに「見積り相談」ボタンを表示`
- `購入前の見積り相談を必須 = ONなら見積り相談経由が前提、OFFなら即購入ボタンも使える`
- `プロフィール経由の見積り受付 = ONならプロフィールからも見積り相談を受けられる`

## 設定の意味（実務向け）
- サービス見積り受付 `ON` + 購入前見積り必須 `OFF`:
見積り相談も即購入も両方可能（現行運用）
- サービス見積り受付 `ON` + 購入前見積り必須 `ON`:
購入前に見積り相談が必要（慎重運用向け）
- サービス見積り受付 `OFF`:
サービスページからの見積り相談は不可（即購入中心）

## このworkspaceの固定運用
- サービス経由見積り受付: `ON`
- 購入前見積り必須: `OFF`
- プロフィール経由見積り受付: `ON`
- プロフィール経由 / メッセージ経由の提案時:
毎回 `出品サービスに紐付ける` を実施

## よくある誤解
- 誤解: `購入してから見積り相談する`
- 実際: 見積り相談は購入前に行う。見積り経由では「提案を購入」する流れになる。
