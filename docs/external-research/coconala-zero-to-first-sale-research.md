# #AR 調査結果: ココナラで実績ゼロから最初の1件を取り、新規出品ブーストを最大活用する戦略

調査日: 2026-04-07
ソース: ココナラブログ、note、個人ブログ、SEO分析サイト（計20+記事を精読）

---

## 1. 現実の数字（知っておくべき事実）

- **出品者の53%が0発注**（ココナラブログ, 2026年3月）
- 1ヶ月で4件成約した出品者は「上位47%」に入る
- 最初の1件が出るまでの平均は **11日〜2ヶ月**（個人差大）
- 閲覧数 3〜5/日、お気に入り0、問い合わせ0 が最初の2週間の「普通」

---

## 2. ココナラの検索アルゴリズム（おすすめ順の仕組み）

### 検索順位を左右する3大要素

1. **直近の販売実績（最重要）**
   - 「直近◯日間の販売額」が基準と言われている（非公開だが複数ソースが一致）
   - 古い実績よりも **直近の勢い** が重視される
   - 1件売れると一気に順位が上がるのはこのため

2. **サービスページの情報充実度**
   - タイトルにキーワードが入っているか
   - 説明文の詳細さ
   - 画像の設定有無（アイキャッチ必須）
   - カテゴリ・タグの適切さ

3. **出品者のアクティブ度**
   - 毎日ログインしているか
   - 返信速度（24時間以内は絶対。早ければ早いほど良い）
   - 最終ログイン日

### 「おすすめ順」と「ランキング順」の違い
- **おすすめ順**: アルゴリズムが総合判断（検索キーワードとのマッチ度 + 実績 + アクティブ度）
- **ランキング順**: 直近の販売件数・金額のみで決まる

## 3. 新規出品ブーストの実態

### ブーストは存在するか？
- ココナラ公式は明言していない
- ただし **複数の出品者が「出品直後に一時的に上位表示された」と報告**
- 「新着」タブに出品直後のサービスが表示される仕組みは確認済み

### ブースト期間の推定
- **出品直後〜7日間** が最も露出が高い（複数ソースが一致）
- 14日を超えると通常のアルゴリズムに乗る
- この間に「1件でも売れる」と、通常アルゴリズムでも浮上する

### ブースト期間にやるべきこと
1. **出品前にサービスページを完璧に仕上げておく**（出品してから直すと遅い）
2. **出品直後から毎日ログイン**（アクティブ度のスコアを上げる）
3. **公開依頼に積極的に提案する**（待ちの姿勢は最大の敵）
4. **ブログ記事を書く**（ココナラブログは内部リンクとして評価される）

---

## 4. 実績ゼロから最初の1件を取る具体戦略

### 戦略A: 公開依頼への提案（最も即効性が高い）

**ココナラの「公開依頼」= 買いたい人が先に依頼を出す機能**

- 実績ゼロでも提案可能
- 提案が選ばれれば、それが最初の実績になる
- **プログラミング系は公開依頼が多い**（バグ修正・コード修正系は特にマッチ）
- 提案文の具体性が決め手。「何を見て、どう進めるか」を書く

#### 提案文のコツ
- ❌「経験豊富です。対応可能です。」
- ✅「〇〇のエラーは△△が原因の可能性が高いです。Stripe の Webhook 設定と署名検証の差を先に確認します。」
- **具体性が全て。** 相手の問題を理解していることを示す

### 戦略B: サービスページの最適化

#### タイトル（最重要。25文字以内推奨）
- キーワードを前に置く
- ベネフィットを明記
- 数字を入れる

```
悪い例: 「プログラミングのバグ直します」
良い例: 「Next.js / Stripeの不具合を診断・修正します【原因特定→修正確認】」
```

#### 説明文
- 冒頭100文字で「何をするか」を書く
- 「こんな方におすすめ」セクション
- 具体的な対応範囲
- 料金と進め方
- 注意事項

#### 画像（アイキャッチ）
- **必ず設定する**（設定なしは致命的）
- キャッチコピーを入れる
- 推奨サイズ: 1280×720px
- Canva等で作成

#### タグ
- 5〜10個設定
- 関連キーワードを入れる
- 「Next.js」「Stripe」「バグ修正」「API」「Webhook」等

### 戦略C: プロフィールの充実

- **アイコン**: 顔写真 or 信頼できるイラスト（空欄は論外）
- **自己紹介**: 何ができるか、どんな経験があるか
- **スキル**: 設定可能なものは全部設定
- **本人確認**: 済ませておく（信頼度UP）

### 戦略D: ココナラブログの活用

- **ココナラ内ブログは検索にプラスに働く**
- 専門性をアピールする記事を書く
- 例: 「Stripe の Webhook が失敗する原因と確認方法」
- サービスページへの内部リンクになる

## 5. やってはいけないこと

1. **出品して「待つ」だけ** — 53%の脱落者の主因
2. **価格を500円にする** — 安すぎると「プロとして見られない」。技術系は適正価格を維持した方がいい
3. **サービスページが未完成のまま出品** — ブースト期間を無駄にする
4. **返信が遅い** — 初回返答時間がプロフィールに表示される。遅いと選ばれない
5. **知人に購入してもらう（サクラ）** — 規約違反のリスク
6. **複数のサービスを大量出品** — 質が落ちて逆効果

## 6. 時系列アクションプラン（出品前〜1件目まで）

### 出品前（準備期間）
- [ ] サービスページの文面を完成させる
- [ ] アイキャッチ画像を作る（1280×720px）
- [ ] プロフィールを充実させる（アイコン、自己紹介、スキル）
- [ ] 本人確認を済ませる
- [ ] ココナラブログの専門記事を1〜2本書いておく

### 出品日（Day 0）
- [ ] 完成したサービスページをそのまま出品
- [ ] 出品直後に公開依頼を確認し、提案できるものに提案する
- [ ] ココナラブログで「サービスを出品しました」系の記事投稿

### Day 1〜7（ブースト期間）
- [ ] 毎日ログインする
- [ ] 公開依頼を毎日チェックし、合うものに提案する
- [ ] 問い合わせが来たら即返信（数時間以内を目指す）
- [ ] 閲覧数・お気に入り数を確認し、タイトルや画像を微調整

### Day 8〜14（ブースト減衰期）
- [ ] 引き続き公開依頼への提案
- [ ] ブログ記事を追加（専門性のアピール）
- [ ] 閲覧数が減ってきたらタイトル・説明文の見直し

### Day 15〜30（通常期）
- [ ] この時点で1件でも実績があれば、通常アルゴリズムで浮上開始
- [ ] 実績がゼロなら、公開依頼への提案を強化
- [ ] SNS（X / note）での告知も検討

## 7. プログラミング系特有のポイント

- **プログラミング系はサービスの「見積もり相談」が多い**（カスタム見積もりで購入される）
- つまり「サービスページを見る → 見積もり相談 → 購入」のフローが主
- **見積もり相談への返信の質が決め手** ← これが返信システムの出番
- 返信が速くて的確だと、そのまま購入される確率が高い
- 逆に返信が遅い / 曖昧だと、他の出品者に流れる

## 9. 返信システムとの接続ポイント

この調査で分かったこと:
- **ココナラで売れるかどうかは「見積もり相談への返信の質」に直結する**
- 返信が速い + 的確 + 具体的 = 購入率UP
- これは今構築した返信システム（4.93/5.0）がそのまま武器になる
- 特に「公開依頼への提案文」の質が最初の1件を決める

→ 返信システムは「待ちの返信」だけでなく、
  「公開依頼への提案文」にも応用できるかもしれない

---

## 10. 今日の調査まとめ

```
最初の1件を取るための優先順位:

  1位: サービスページを完璧にしてから出品する（ブースト期間を無駄にしない）
  2位: 公開依頼に毎日提案する（待ちの姿勢は最大の敵）
  3位: 毎日ログインする（アクティブ度スコア）
  4位: 見積もり相談への即レス + 的確な返信（返信システムの出番）
  5位: ココナラブログで専門記事を書く（SEO + 内部リンク）

  やらなくていいこと:
  - 値下げ（15,000円を維持してOK）
  - 大量出品
  - SNS集客（最初は不要。ココナラ内で完結できる）
```
