## 理論1: ココナラ固有のサムネイル仕様

**出典: サムネAI（2026）、ヨルダケデザインブログ（2025）、ココナラブログ100件分析**

### サイズ仕様

```
ココナラ推奨サイズ:
  - 1220 × 1240px（ほぼ正方形、わずかに縦長）
  - 解像度: 72〜150ppi

表示サイズ（buyer が実際に見るサイズ）:
  - 検索結果カード: 約150px幅（← ここで勝負が決まる）
  - サービスページ: 大きく表示
  - スマホ: PC より小さく表示

→ つまり、1220px で作ったものが 150px に縮小される
→ 1/8 サイズで読めるかどうかがすべて
```

### 100件分析から導かれた法則（ココナラブログ/チハル）

```
クリックされるサムネの共通点:
  1. キャッチコピーが1行で完結（最大2行）
  2. テキスト情報は最大3要素まで
  3. 太めのゴシック体を使用
  4. 背景とテキストのコントラストが高い
  5. 競合と違う色を使っている

クリックされないサムネの共通点:
  1. 文字を詰め込みすぎ（3行以上のテキスト）
  2. 画質が悪い / ぼやけている
  3. テキスト情報がなさすぎ（写真だけ）
  4. 他の出品者と見分けがつかない
  5. スマホで文字が読めない
```

### 今の画像はどちらに当てはまるか

```
✅ テキスト情報は3要素（行1/行2/行3）← OK
❌ 行1と行3が小さすぎて150px幅で読めない ← NG
❌ 2枚の画像がほぼ同じ色 ← NG（差別化できない）
✅ テキスト情報はある（写真だけではない）← OK
⚠️ フォントがゴシック体だが太さが足りない可能性 ← 要確認
```

## 理論2: タイポグラフィの科学（小さい画面で読ませる）

**出典: ThumbnailCreator タイポグラフィガイド（2026）、テキストコントラストガイド（2026）**

### 鉄則

```
1. フォント: 太いサンセリフ体一択
   → おすすめ: Noto Sans CJK JP Bold、M PLUS Rounded 1c Bold、Impact
   → 明朝体・細いフォント = NG（150px幅で消える）

2. 文字数: メインテキストは3-5語（日本語なら6-8文字）
   → 理由: 文字数が増えるほどフォントサイズが下がる
   → 今の「不具合診断・修正します」は8文字 → ギリギリOK
   → 今の「現状整理・引き継ぎメモ」は10文字 → やや多い

3. コントラスト: 4-8px のアウトライン or 背景とのコントラスト比 4.5:1 以上
   → 今の画像: メインテキスト（白 on ダークネイビー）→ OK
   → 今の画像: 行1（グレー on ダークネイビー）→ NG（コントラスト不足）

4. テスト: 168×94px まで縮小して読めるか確認（squint test）
   → 今の画像を150px幅に縮小 → 行1と行3 は確実に読めない

5. モバイルファースト: ユーザーの大半はスマホで閲覧
   → ココナラも同様。スマホの検索結果で判断される
```

### 研究データ
```
ThumbnailCreator（2025年10月研究）:
  - よりlegibleなフォントに変更 → CTR（クリック率）23%向上
  - カスタムサムネイルを使用 → YouTube上位の90%が使用

適用:
  → ココナラでもサムネの質がクリック率に直結する
  → 特に「検索結果で並んだ時の可読性」が最重要
```

## 理論3: マーケットプレイスのサムネイル設計（Fiverr参考）

**出典: Fiverr公式ヘルプセンター、Touhfa Art（2026）、Jobbers（2026）**

### Fiverr の推奨

```
サイズ: 1280 × 769px
ルール:
  - テキストは大きく、背景とのコントラストを高く
  - 「何のサービスか」が0.5秒で分かること
  - カスタム画像は stock photo より CTR が高い
  - 余白を適切に使う（ただし50%空白はNG）

Fiverr で売れてるサムネの共通点:
  1. 色が鮮やか（ダーク一色ではない）
  2. テキストが大きい（画像の40-60%をテキストが占める）
  3. アクセントカラーが入っている（価格や特徴を強調）
  4. 「何をしてくれるか」が1文で分かる
```

### ココナラへの適用

```
Fiverr とココナラの違い:
  - Fiverr は横長（16:9）、ココナラはほぼ正方形（1:1）
  - ココナラの方がテキスト配置の自由度が高い
  - ただし、検索結果でのカード表示サイズは両方とも小さい

共通して使える原則:
  - テキストを大きく
  - コントラストを高く
  - 色で差別化
  - 「何のサービスか」を0.5秒で理解させる
```

## 理論4: 色彩心理学とサムネイルの色戦略

**出典: サムネAI（2026）、ThumbnailCreator（2026）、100件分析**

### ココナラ IT系の色傾向

```
IT系サービスの出品者が使いがちな色:
  - ダークネイビー / ダークブルー ← 今の画像がまさにこれ
  - ブラック
  - グレー

つまり:
  → IT系の検索結果は「暗い画像だらけ」になる可能性が高い
  → 暗い中に暗い画像を出しても埋もれる

差別化戦略:
  1. 競合と違う色を使う
  2. アクセントカラーで目立つ要素を入れる
  3. 白背景 + 濃い文字 という逆張りも有効
```

### 色の心理効果

```
ネイビー/ダークブルー: 信頼、専門性、落ち着き → 今の画像
グリーン: 安心、解決、成長 → bugfix に合う
オレンジ/黄色: 注意、警告、緊急 → bugfix の「不具合」に合う
白: 清潔、シンプル、プロフェッショナル → Quiet Luxury に合う

推奨:
  → 全面的に色を変える必要はないが、
    アクセントカラー1色を追加することで差別化できる
  → bugfix: 現在のネイビー + アクセントに緑 or オレンジのワンポイント
  → handoff: 背景色を少し変えて bugfix と差別化
```

## 理論5: 「3要素ルール」（情報設計）

**出典: サムネAI（2026）、Fiverr最適化ガイド（2026）**

### 核心

```
サムネイルに入れる情報は最大3つ:

  1. キャッチコピー（最優先）← 何のサービスか
  2. 信頼要素 ← 実績数、技術名、資格など
  3. ターゲット or メリット ← 誰向けか、依頼するとどうなるか

今の画像の3要素:
  bugfix:
    1. 不具合診断・修正します ← キャッチコピー ✅
    2. Next.js / Stripe ← 信頼要素（技術名）✅
    3. Webhook・API連携も対応 ← メリット ✅

  handoff:
    1. 現状整理・引き継ぎメモ ← キャッチコピー ✅
    2. AI / 外注コード ← ターゲット ✅
    3. まず現状を整理 ← メリット ⚠️（弱い。繰り返しに感じる）

→ 3要素の選定は概ね正しい
→ 問題は「サイズ」と「コントラスト」であって「内容」ではない
```

## Codex への提案: サムネイルをどう改善すべきか

### 大前提
```
今の画像の「内容」（テキスト要素の選定）はほぼ正しい。
問題は「サイズ」「コントラスト」「差別化」の3点。
全く別のデザインにする必要はない。
Quiet Luxury のトーンは維持しつつ、可読性を上げる。
```

### 提案の優先順位

| 優先度 | 提案 | 変更量 | 期待効果 |
|---|---|---|---|
| ★★★ | メインテキストを画像面積の40-50%に拡大 | レイアウト変更 | 150px幅で読めるようになる |
| ★★★ | 下半分の空白を活用（テキストを中央に配置） | レイアウト変更 | 情報密度の向上 |
| ★★☆ | 行1（技術名）のコントラストを上げる（白に近づける） | 色変更 | 副テキストの可読性向上 |
| ★★☆ | 2枚の画像の色味を差別化する | 色変更 | 並べた時に区別がつく |
| ★☆☆ | アクセントカラーの追加（1色） | 色追加 | 検索結果での視認性向上 |

### 提案1: テキストサイズの拡大（★★★）

```
現在:
  テキストが画像の上半分に偏り、下半分が空白
  メインテキスト（「不具合診断・修正します」）が画像面積の約20-25%

改善後:
  テキストを垂直方向で中央寄せ
  メインテキストを画像面積の40-50%に拡大
  行1（技術名）と行3（補足）のフォントサイズも比例して拡大

具体的なサイズ目安（1220×1240px画像で）:
  メインテキスト: 100-120px → 160-200px に拡大
  行1: 40-50px → 60-70px に拡大
  行3: 30-40px → 50-60px に拡大

テスト方法:
  完成後、画像を150px幅に縮小してスマホで確認
  「squint test」: 目を細めてもメインテキストが読めるか
```

### 提案2: 2枚の色差別化（★★☆）

```
現在: 両方ともダークネイビー

改善案:
  bugfix-15000: ダークネイビー維持（現状のまま）
  handoff-25000: やや明るいダークグレー or ダークティール

  → 微妙な色差でも、並んだ時に「違うサービスだ」と認識される
  → 大胆に変える必要はない（Quiet Luxury を維持する）
```

### 提案3: アクセントカラー（★☆☆）

```
現在: ネイビー + 白 + グレー + 青 の4色（全て寒色系）

追加候補:
  bugfix: 薄い緑のアクセント（「解決」「修正完了」のイメージ）
    → 行3のテキスト色を薄い緑に変更
    → または、「15,000円」をアクセントカラーで表示

  handoff: 薄いオレンジ or アンバーのアクセント（「整理」「可視化」のイメージ）
    → 行3のテキスト色をアンバーに変更

注意: アクセントは1色のみ。2色以上入れるとゴチャつく。
```

### 提案4: handoff の行3 テキスト変更（★☆☆）

```
現在: 「まず現状を整理」← 行2 の「現状整理」と重複感
改善案: 「危険箇所と次の1手を整理」← サービスの本質を示す
     または: 「次に触る人が迷わない状態に」← Transformation
```

### 画像サイズ仕様の確認

```
ココナラ推奨: 1220 × 1240px（ほぼ正方形）
現在の画像: 確認が必要（正方形に見えるが実寸は不明）
→ Codex に確認してもらう

注意: ココナラはサムネイルをトリミングする場合がある
→ 重要な要素は画像の中央70%以内に配置する（安全余白）
```

### 今回変えてはいけないこと

| 項目 | 理由 |
|---|---|
| Quiet Luxury のトーン | プロフィール戦略と一貫性を保つ |
| 幾何学パターン（六角形） | 技術系の専門性を暗示する良い要素 |
| テキストの内容（3要素の選定） | 概ね正しい。サイズだけの問題 |
| ダークカラー基調 | IT系の信頼感を維持 |
