# ココナラ プラットフォーム完全ガイド（出品者視点 + 購入者視点）

## 調査目的
Codex が返信を書く時に「ココナラ自体の仕組み」を正確に理解していること。
これにより、buyer の質問に対して「ココナラのプラットフォーム仕組み」を踏まえた返信ができる。

## 1. 全体の取引フロー（出品者視点）

```
Phase 1: 出品
  → 出品フォームからサービス内容を登録し公開
  → サービスページが公開される

Phase 2: 購入前相談（prequote）
  ├── メッセージ機能（旧ダイレクトメッセージ）で buyer と事前にやりとり
  ├── 「見積もり相談」で buyer が相談を送ることもできる
  └── ★ メッセージ機能でやりとりしていても、まだ「取引」は始まっていない

Phase 3: 購入
  ├── buyer がサービスを購入 or 見積もり提案を承認して購入
  ├── → 非公開のトークルームが自動で開く ★
  ├── → 48時間以内に出品者から初回連絡が必要 ★★★
  └── メッセージ機能とトークルームは別物 ★★

Phase 4: 取引中（purchased）
  ├── トークルームでやりとり
  ├── 要件の確認・ヒアリング
  ├── 作業を進める
  ├── 追加支払いが必要な場合 → 「おひねり」or「有料オプション」
  ├── こまめな進捗報告 ★
  └── 出品者の最終連絡から4日以上放置 → buyer がキャンセルリクエスト可能 ★★

Phase 5: 納品
  ├── 「正式な納品」を送信 → トークルームをクローズする手続き
  ├── buyer に「納品確認」が届く
  ├── buyer が「承諾」→ 24時間後にトークルームクローズ
  ├── buyer が「差し戻し」→ トークルームで追加のすり合わせ ★
  ├── ★ 差し戻しは1回のみ
  └── ★ 2回目の正式な納品後、3日でトークルームが自動クローズ

Phase 6: 評価
  ├── 出品者 → 購入者を評価（任意）
  ├── 購入者 → 出品者を評価（任意）★公開される
  ├── 評価コメント（全体評価・良かった点）は公開
  ├── 5段階評価の「総合評価」のみ公開
  └── 評価入力がなくても3日後に取引完了 + 売上計上

Phase 7: 取引完了
  ├── トークルームクローズ
  ├── 売上がマイページに反映
  └── トークルームクローズ後にやりとりしたい場合 → メッセージ機能を使う
```

## 2. 購入の経路（buyer 視点）

```
経路A: サービスページから直接購入
  → サービスページを見る → 「購入画面に進む」→ 決済 → トークルーム open

経路B: 見積もり相談から購入
  → サービスページの「見積もり・カスタマイズの相談」ボタン
  → buyer が相談内容を入力して送信
  → 出品者が「見積もり提案」を返す（金額・内容・納期を提案）
  → buyer が提案を「承認」して購入 → トークルーム open

経路C: メッセージで事前相談してから購入
  → 出品者のプロフィール or サービスページからメッセージを送る
  → やりとりして合意 → buyer がサービスページから購入 → トークルーム open
```

---

## 3. メッセージ機能 vs トークルーム（★最重要★）

```
メッセージ機能:
  - 購入前のやりとりに使う
  - 取引完了後のアフターフォローにも使える
  - 購入していなくても使える
  - ファイル添付可能
  - ★ ここでやりとりしても「取引」は始まっていない
  - ★ 購入後にメッセージでやりとりしても、トークルームの48時間ルールは別

トークルーム:
  - 購入後に自動で開く非公開の取引空間
  - ファイル添付可能
  - ★ ここが「取引」の場所
  - ★ 出品者の最終連絡から4日以上放置でキャンセルリクエスト可能
  - ★ 正式な納品はここから送る
  - ★ トークルームクローズ = 取引完了
```

---

## 4. 見積もり提案の仕組み

```
buyer 側の操作:
  1. サービスページの「見積もり・カスタマイズの相談」をクリック
  2. 相談内容を入力して送信
  3. 出品者からの見積もり提案を待つ
  4. 提案を見て「購入に進む」or「辞退する」

出品者側の操作:
  1. 見積もり相談が届く
  2. 相談内容を確認
  3. 「見積もり提案を送る」で金額・内容・納期を入力
  4. 送信 → buyer に見積もり提案が届く

★ 見積もり提案 = サービスの金額をカスタマイズできる
  → 有料オプションを付けた提案もできる
  → 追加フロー（15,000円）を含む提案もここで出す
```

## 5. 正式な納品と納品確認

```
正式な納品:
  - 出品者がトークルームから送信する
  - 「正式な納品を送ってクローズする」チェックボックス（PC）
  - 赤い「正式な納品」ボタン（アプリ）
  - ★ 送信前に buyer に「送って問題ないか」確認すべき
  - ★ 送信後は追加料金を請求できなくなる

納品確認（buyer 側）:
  - buyer が受け取った納品物を確認
  - 「承諾」→ 24時間後にトークルームクローズ
  - 「差し戻し」→ 出品者と追加のすり合わせ
  - ★ 差し戻しは1回のみ
  - 何も選択しない → 72時間後に自動クローズ

トークルームクローズ後:
  - 売上が計上される
  - もうメッセージは送れない
  - やりとりしたい場合はメッセージ機能（DM）を使う
```

---

## 6. キャンセルの仕組み

```
出品者側からキャンセル:
  - 購入者の連絡が途絶えた場合、出品者からキャンセルリクエスト可能
  - buyer が24時間以内に応答しなければ自動キャンセル

buyer 側からキャンセル:
  - 出品者の最終連絡から4日以上経過 + 最終送信者が buyer
  - → キャンセルリクエスト送信可能
  - 出品者が24時間以内に返信しなければ自動キャンセル + 星1

双方合意のキャンセル:
  - 両方が合意 → キャンセル成立 → 評価は付かない
  - 全額返金される

★ 返金や減額は、支払った経路やタイミングで扱いが分かれる
  → おひねり / トークルームオープン後の有料オプションは、双方合意のうえ運営取消になるケースがある
  → サービス本体や同時購入オプションの一部減額は、再購入 + 既存トークルームのキャンセルで調整する案内が公式ヘルプにある
  → 途中キャンセルでは、出品者がこの場で返金可否や金額を断定しない方が安全

★ 途中まで作業した分の補償は、ココナラの仕組みでは保証されない
  → 作業着手前にスコープと条件を明確にしておくことが最重要
```

## 7. おひねり（追加支払い）

```
おひねりとは:
  - 追加で料金を支払える機能
  - buyer がトークルーム内から送信
  - 100円〜の任意の金額

使われるケース:
  1. 追加作業が発生した場合の追加料金（出品者がお願いする）
  2. buyer が感謝の気持ちとして自発的に送る

★ 正式な納品後でも、クローズ前の一定期間はおひねりを支払える
  → ただし、どのタイミングで支払われた金額かによって取消方法が変わる
  → 実務上は、追加料金の合意を正式な納品前に終えておく方が安全
```

---

## 8. 有料オプション

```
有料オプションとは:
  - サービスに追加できるオプション（出品者が設定）
  - 例: 「追加1フロー +15,000円」
  - buyer が購入時に選択 or トークルーム内で追加購入

★ 見積もり提案にオプションを含めることも可能
```

---

## 9. 評価システム

```
構成:
  - 5段階の星評価（総合評価のみ公開）
  - 評価コメント（良かった点。公開される）
  - やりとり評価は非公開

タイミング:
  - 正式な納品 → buyer が承諾 → 評価入力期間
  - 評価入力は任意
  - 相手の評価が見えるのは、自分も評価を入力した後（相互公開）

影響:
  - 平均評価 → 検索順位 + 出品者ランクに影響
  - 納品完了率（キャンセル率）→ ランクに影響
  - 出品者ランク: レギュラー → ブロンズ → シルバー → ゴールド → プラチナ
```

## 10. 手数料

```
ココナラの手数料:
  - サービス価格の 22%（税込）
  → 15,000円のサービス → 手取り約 11,700円
  → 25,000円のサービス → 手取り約 19,500円

★ 価格設定時にこの手数料を考慮する必要がある
```

## 11. 返信システムへの反映ポイント

### 致命的（知らないと事故になる）

```
1. 48時間ルール → purchased 初回は48h以内に必ずトークルームで返す
2. 4日間ルール → 4日以上放置しない。催促前に自発報告
3. メッセージ ≠ トークルーム → 購入後はトークルームで返す
4. 正式な納品 = 取引終了の開始 → buyer に確認してから送る
5. 差し戻しは1回のみ → 2回目の正式な納品で3日後に自動クローズ
6. 外部通信禁止 → Zoom/LINE/Skype は双方NG
7. おひねり/追加料金 → 正式な納品後でもクローズ前に支払える時間帯はあるが、実務上は正式な納品前に合意を終える
```

### 品質向上（知っていると返信が良くなる）

```
1. buyer が「見積もり相談」から来た場合
   → 「見積もり提案」で金額・内容・納期を返す操作があることを知る
   
2. buyer が「修正してほしい」と言った場合
   → 正式な納品前なら通常のやりとりとして対応
   → 正式な納品後の差し戻しは1回のみ
   
3. buyer が「キャンセルしたい」と言った場合
   → 返金や減額は支払い経路とタイミングで扱いが分かれる
   → 双方合意のキャンセルなら評価は付かない
   → 「ココナラの仕組みに沿って確認します」が正しい
   
4. buyer が「次も使いたい」と言った場合
   → リピート購入は通常のサービス購入を案内
   → 取引完了後はメッセージ機能で連絡可能
   
5. 追加作業が発生した場合
   → おひねり or 有料オプションで対応
   → 正式な納品の前に支払いを確認
```
