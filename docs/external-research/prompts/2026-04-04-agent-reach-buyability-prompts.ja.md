# Agent-Reach Prompt 集

目的:
- `#AR` で後からそのまま使える、売れやすさ改善向けの調査 prompt をまとめておく
- 今すぐ実行しないものも、外部調査レーンのメモとして退避する
- 正本ではなく、調査の入口として使う

使い方:
- `#AR` の後に、そのまま貼って使う
- 返ってきた調査結果は `docs/external-research/` 配下に日付つきメモとして保存する
- 結果をそのまま正本へ入れず、Codex が選別してから反映する

## 1. 痛み語調査

```text
#AR
ココナラや周辺実務記事を横断して、Next.js / Stripe / API連携の不具合で依頼者が実際に使う「困りごとの言い方」を集めてください。
目的は、サービスページと返信文で「買い手が自分ごと化しやすい症状語」を特定することです。

見たいもの:
- 症状語
- 不安語
- 急ぎ案件で使われやすい表現
- 非エンジニアが使う言い方
- エンジニアが使う言い方

出力形式:
1. よく出る症状語 上位20
2. 相談文に出やすい不安語 上位10
3. bugfix-15000 の冒頭に入れると刺さりやすい言い回し候補 10 個
4. 逆に使わない方がよい抽象語
5. 参照URL
```

## 2. 競合の冒頭3行調査

```text
#AR
ココナラ上で、Stripe / Webhook / API連携 / 外注コード修正 / 引き継ぎコード調査に近いサービスを集めてください。
目的は、「購入されやすそうなサービスが冒頭3行で何を約束しているか」を見ることです。

見たいもの:
- タイトル
- 冒頭3行
- 誰向けか
- 何を含むと見せているか
- 何を安心材料にしているか

出力形式:
1. 競合サービス 10件
2. 各サービスの冒頭3行要約
3. 強い共通パターン
4. 弱い共通パターン
5. bugfix-15000 に持ち込める要素
6. handoff 系に持ち込める要素
7. 参照URL
```

## 3. 購入前の離脱ポイント調査

```text
#AR
ココナラのサービスページ、比較記事、出品ノウハウ、レビュー観点を横断して、「購入前に止まりやすい理由」を調べてください。
目的は、サービスページや見積り返信で何が摩擦になっているかを特定することです。

前提:
- 現在の主力は 15,000円の bugfix サービス
- 強みは Next.js / Stripe / API連携
- 課題は「自分が対象か分からない」「整理だけに見える」「相談前の負担感」

出力形式:
1. 購入前の離脱理由 上位10
2. 価格ではなく説明不足で止まる典型例
3. 情報要求が重く見える典型例
4. bugfix-15000 の離脱要因になりやすい点
5. handoff 系の離脱要因になりやすい点
6. すぐ直せる改善案 上位5
7. 参照URL
```

## 4. 次サービス候補調査

```text
#AR
現在の bugfix-15000 と相性がよく、同じ顧客の次の悩みとして成立しやすいサービス候補を調べてください。
目的は、「今の受注導線から自然につながる次サービス」を見つけることです。

前提:
- 現在は Next.js / Stripe / API連携の不具合修正が主力
- 外注コード、AI生成コード、前任者不在の文脈と相性がよい
- ただし、机上のアイデアではなく、実際に需要が見えるものを優先したい

出力形式:
1. 有望な次サービス候補 10個
2. 各候補の想定痛み
3. bugfix サービスから自然につながる理由
4. 差別化しやすい候補 上位5
5. 今はやらない方がよい候補
6. 参照URL
```

## 5. 購入率を上げる表現調査

```text
#AR
ココナラ、周辺実務記事、SaaSサポート事例を横断して、「相談しやすく見える表現」「購入しやすく見える表現」を集めてください。
目的は、サービスページと返信文の両方で使える“購入の背中を押す表現”を見つけることです。

見たいもの:
- 安心材料の出し方
- 買い手の負担を下げる言い方
- 専門性を出しつつ重く見せない言い方
- 範囲を守りつつ断りすぎに見えない言い方

出力形式:
1. 購入率に効きやすい表現パターン 15個
2. 負担を下げる言い方 10個
3. 専門性が伝わるが重くない言い方 10個
4. 逆に購入率を落としやすい表現 10個
5. bugfix-15000 の冒頭文に使える例文 5本
6. 見積り返信の初手に使える例文 5本
7. 参照URL
```

3. 購入前の離脱ポイント調査
4. 購入率を上げる表現調査
5. 次サービス候補調査
