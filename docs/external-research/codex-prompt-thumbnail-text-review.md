## 調査から分かったルール

### 3要素ルール（サムネAI / Fiverr最適化ガイド）
```
サムネイルに載せる情報は最大3つ:
  1. キャッチコピー（最優先）← 何のサービスか。行2 に相当。
  2. 信頼要素 ← 技術名、実績、対象範囲。行1 に相当。
  3. ターゲット or メリット ← 誰向けか、頼むとどうなるか。行3 に相当。
```

### タイポグラフィの制約（ThumbnailCreator 2026）
```
- メインテキスト（行2）: 日本語で 6-8文字が限界
  → これ以上長いとフォントサイズが下がって 150px幅で読めなくなる
- 行1・行3: 補助なので短いほどいい
- 全体で情報を詰め込みすぎない
```

### 150px幅テスト
```
ココナラの検索結果カードは約150px幅で表示される
→ 1220pxで作ったものが1/8に縮小される
→ 「行2だけでも読めるか」が最低ライン
→ 行1・行3は「読めれば加点、読めなくてもOK」
```

## Claude の気づき

### bugfix-15000 について
```
行1「Next.js / Stripe」:
  → 技術名を信頼要素として使っている。良い。
  → ただし buyer が技術名を知らない場合もある（非技術者）

行2「不具合診断・修正します」:
  → 8文字。ギリギリOK。
  → 「何をしてくれるか」が明確。

行3「Webhook・API連携も対応」:
  → 技術者向けには刺さるが、非技術者には意味不明
  → 検索結果で読めないなら、買い手に届くメッセージに変えた方がいいかも
  → 候補: 「原因不明でも対応」「まず症状だけでOK」
```

### handoff-25000 について
```
行1「AI / 外注コード」:
  → ターゲット指定。良い。buyer が「自分のことだ」と思える。

行2「現状整理・引き継ぎメモ」:
  → 10文字。やや長い。150px幅で読めるか不安。
  → 候補: 「主要1フローを整理」（8文字）
  → 候補: 「コード整理・引き継ぎ」（9文字）

  → 行2 の「現状整理」と重複。情報量が増えていない。
  → 候補: 「危険箇所と次の1手」← サービスの核心を示す
  → 候補: 「次に触る人が迷わない状態に」← Transformation（長い）
```

## Codex に判断してほしいこと

### 1. bugfix-15000 のテキスト3行
```
行1: 「Next.js / Stripe」のままでよいか
行2: 「不具合診断・修正します」のままでよいか
行3: 「Webhook・API連携も対応」のままでよいか
     → 非技術者には意味不明。変えるべきか？
     → 変えるなら何にするか
```

### 2. handoff-25000 のテキスト3行
```
行1: 「AI / 外注コード」のままでよいか
行2: 「現状整理・引き継ぎメモ」のままでよいか
     → 10文字で長い。短縮すべきか？
行3: 「まず現状を整理」は行2との重複。変えるべきか？
     → 変えるなら何にするか
```

### 3. 2枚の統一性

```
bugfix と handoff の行2 は、並んだ時に:
  - 「別のサービスだ」と一瞬で区別がつくか
  - でも「同じ出品者だ」と分かる統一感があるか
```

### 4. 守ってほしい条件

```
- 固定条件（価格・範囲など）を画像テキストに入れてはいけない
  （価格はココナラUIが自動表示するため）
- サービスタイトル（25字以内）との整合性を取る
  → bugfix: 「Next.js/Stripe不具合診断・修正します」
  → handoff: 「AI/外注コードの主要1フローを整理します」
- 日本語として自然であること
- 営業感を出さないこと
```
