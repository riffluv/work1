# ココナラ プラットフォーム不文律・ビジネスマナー調査

## 調査目的
返信システムに「ココナラ特有のルール・不文律」を knowledge として入れるべきかを判断するため。

## ソース
- ココナラマグ公式（mag.coconala.com）
- 実務経験者ブログ（300件以上の取引実績者）
- ココナラ利用規約

---

## 1. 致命的ルール（違反すると星1 or アカウント停止）

### 48時間ルール
- **購入後48時間以内に出品者からトークルームで初回連絡がないと、自動キャンセル**
- 自動で ★☆☆☆☆（星1）が付く
- 全サービスが受付休止中になる
- → 返信システムへの影響: **購入通知を受けたら48時間以内に必ず返信する必要がある**

### 4日間ルール（出品者連絡なしキャンセル）
- 出品者の最終連絡から4日以上経過 + 最終メッセージ送信者が購入者
- → 購入者からキャンセルリクエスト可能
- 24時間以内に出品者から返信がなければ自動キャンセル + 星1
- → 返信システムへの影響: **purchased 中に4日以上放置しない。催促が来る前に進捗を送る**

### 外部やりとり禁止
- Zoom / LINE / Skype への誘導は **双方とも禁止行為**
- ただし、ココナラ内の「ビデオチャット」は一部カテゴリで利用可能
- → 返信システムへの影響: **hard_no に「Zoom / 通話」は正しかった。ただし「ココナラのビデオチャット」という選択肢があることを知っておくべき**
- ※ただしIT・プログラミングカテゴリでビデオチャットが使えるかは要確認

### 外部リンク禁止
- 自サイトなど外部へのリンク記載は禁止
- ポートフォリオもココナラ内に置く
- → 返信システムへの影響: **返信文に外部URLを貼らない**

### 直接取引禁止
- ココナラ外での決済・取引は禁止
- 「別のところで直接お願いできますか？」は断る必要がある

## 2. 評価に関する不文律

### 評価コメントの重要性
- 星評価は今後の検索順位とランクに直結
- 300件以上の取引経験者によると:
  - 納品完了率が一定ラインを下回るとランクダウン
  - キャンセル数も納品完了率に影響
  - 評価がもらえないケースもある（リピーターでも）

### 低評価を避けるポイント（実務経験者の知見）
- **認識ズレを防ぐ**: サービスページの内容と実際の対応を一致させる
- **こまめな進捗報告**: 連絡が途絶えると不安→低評価の原因
- **スケジュールの事前共有**: 納品予定日だけでなくプロセスを見える化
- **期待値の調整**: 購入前に「何ができて何ができないか」を明確にする

## 3. キャンセルに関するルール

### 出品者からのキャンセル
- 出品者からもキャンセルリクエストを送れる
- 購入者都合のキャンセルの場合、出品者には影響が少ない
- ただし、出品者都合のキャンセルは納品完了率に影響

### 途中キャンセルの扱い
- 返金や減額は、支払った経路とタイミングで扱いが分かれる
- おひねり / トークルームオープン後の有料オプションは、双方合意のうえ運営取消になるケースがある
- サービス本体や同時購入オプションの一部減額は、再購入 + 既存トークルームのキャンセルで調整する案内が公式ヘルプにある
- → 返信システムへの影響: **CASE 10（途中キャンセル）の返信で「返金はココナラの仕組みに沿って対応します」が正しい**

### キャンセルと評価の関係
- 双方合意のキャンセル → 評価は付かない
- 出品者連絡なしキャンセル → 自動で星1
- 一方的なキャンセルリクエスト → 状況による

---

## 4. トークルームの不文律

### メッセージとトークルームの使い分け
- **メッセージ**: 購入前のやりとり（見積もり相談）
- **トークルーム**: 購入後のやりとり
- メッセージでやりとりしていても、トークルームで48時間以内に連絡しないと自動キャンセル
- → 返信システムへの影響: **purchased になったら、メッセージではなくトークルームで返す。reply memory はトークルーム前提**

### 進捗報告のマナー
- 長期間の作業は定期報告が必要
- ずれる場合は「ずれる前に」早めに連絡
- → 返信システムへの影響: **deadline を破る時は、破る前に連絡。破った後の謝罪だけでは不十分**

## 5. 返信システムへの反映推奨

### 即時反映すべき knowledge（致命的）

```
coconala_rules:
  purchase_response_deadline: 48h
  no_response_cancel_threshold: 4days
  external_communication: prohibited  # Zoom/LINE/Skype
  external_links: prohibited
  direct_transaction: prohibited
  cancellation_refund: "ココナラの仕組みに沿って対応"
```

### 反映を検討すべき knowledge（品質向上）

```
coconala_best_practices:
  progress_report: "長期作業は定期報告。ずれる前に早めに連絡"
  schedule_sharing: "納品予定日だけでなくプロセスを見える化"
  expectation_management: "何ができて何ができないかを購入前に明確に"
  video_chat: "ココナラ内のビデオチャットは一部カテゴリで利用可能（要確認）"
```

### 既に対応できていること
- ✅ Zoom / 通話の拒否 → hard_no に入っている
- ✅ 直接取引の拒否 → hard_no に入っている
- ✅ 進捗報告 → reply memory の deadline 管理
- ✅ 期待値調整 → service.yaml の scope 定義

### まだ対応できていない可能性があること
- ⚠️ 48時間ルールの明示的な認識
- ⚠️ 4日間ルール（催促前に自発的に連絡）
- ⚠️ キャンセル時の返金ルール（ココナラの仕組み準拠）
- ⚠️ ビデオチャット機能の存在（IT カテゴリで使えるか）
- ⚠️ 外部リンク禁止の明示
