# 初案件 受注〜納品 1ページ手順

更新日: 2026-02-28

## 1. 相談受付（受注前）
- 受領メッセージでテキスト対応・標準範囲外を明示する
- ヒアリング項目を回収する
- 回収必須項目は以下の6点
1. 期待する動作
2. 実際の挙動
3. 再現手順
4. エラーログ（機密マスク）
5. スタック・ランタイム・デプロイ先
6. 希望納期

## 2. スコープ確定（受注前）
- 基本料金15,000円の対象を確認する（不具合1件）
- 不具合1件の定義を合意する（同一原因 / 1フロー / 1エンドポイント）
- 追加対応が必要な場合は有料オプションで対応することを明記する
- 範囲外を再掲する（通話なし、直接pushなし、本番デプロイなし）
- スコープ変更時は作業停止して別見積を提示する

## 3. 診断開始（受注後）
- 症状を再現し、再現条件を記録する
- 仮説を立て、ログ・コード・設定を順に検証する
- 再現不可なら「不足データ一覧」を先に提出する

## 4. 修正対応（基本範囲）
- 変更は最小単位で実施する
- 変更対象ファイルと変更理由を記録する
- 可逆性を維持する（小さな差分、説明可能な手順）

## 5. 検証
- 修正前に失敗していた手順で再試験する
- 正常系と副作用リスクを確認する
- 検証結果を手順付きで文書化する

## 6. 納品
- `00_結論と確認方法.md` を納品する（全案件共通）
- 修正済みファイルをZIPで納品する（標準）。必要時のみpatch/diffを補助添付する
- 納品文に「何が壊れていたか / 何を変えたか / どう確認するか」を明記する

## 7. クローズ
- ケースを `cases/CLOSED` へ移動する
- `CASES_INDEX.md` を更新する
- 次回改善メモ（再発防止観点）を1行残す
