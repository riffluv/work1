# ココナラ書類依頼時の運用メモ

## 用途
- 購入者から `見積書` `納品書` `請求書` `契約書` の相談が来た時の即応用
- Codex に `見積書を求められている` と伝えた時に、このファイル基準で判断する

## 基本方針
- まず `法人機能利用ユーザーか / 個人アカウントか` を確認する
- 法人機能利用ユーザーなら、まずココナラ標準の書類発行機能を案内する
- 個人アカウントなら、出品者判断で対応する
- 書類送付は `トークルーム` または `メッセージ` の添付で行う
- 外部メール送付や外部共有はしない

## 公式ルール要点
- 購入者が法人機能利用ユーザーの場合:
  - ココナラの各種書類発行機能を利用するよう案内する
- 購入者が個人アカウントの場合:
  - 出品者判断で対応
  - 自前フォーマットまたはココナラ公式フォーマットを利用してよい
- 書類に電話番号やメールアドレスは記載しない
- 発行元名:
  - 法人は法人名
  - 個人は本名またはビジネスネーム
- 住所は任意
  - 必要時のみ、相手と合意した内容を記載

## 判断フロー
1. 相手が何を求めているか確認
   - 見積書
   - 納品書
   - 請求書
   - 契約書
2. `法人機能利用ユーザーかどうか` を確認
3. 法人機能利用ユーザーなら:
   - まずココナラ標準機能を案内
4. 個人アカウントなら:
   - 発行するか判断
   - 必要なら公式テンプレを使う
5. 記載前に確認
   - 発行元名
   - 宛名
   - 金額
   - 日付
   - 住所記載要否
6. 禁止事項確認
   - 電話番号なし
   - メールアドレスなし
   - 外部送付なし

## 実務判断
- 見積書:
  - 基本は対応可能
  - 見積り相談の金額・内容と一致させる
- 納品書:
  - 基本は対応可能
  - 正式な納品内容と一致させる
- 請求書:
  - まず相手が法人機能利用ユーザーか確認
  - 個人アカウント相手で必要性が高い場合は、発行可否を都度判断
- 契約書:
  - その場で安請け合いしない
  - 内容確認のうえ、対応可否を判断

## こちらの返答テンプレ

### 1. 法人機能ユーザー向け
```text
ご連絡ありがとうございます。
法人機能をご利用の場合は、ココナラの書類発行機能をご利用いただく形がスムーズです。
まずはそちらをご確認いただけますか。
もし追加でこちらの記載内容確認が必要でしたら、その範囲で対応します。
```

### 2. 個人アカウント相手で対応する場合
```text
ご連絡ありがとうございます。
見積書 / 納品書の発行は対応できます。
ココナラのトークルーム添付でお送りしますので、宛名と必要な書類名を教えてください。
必要であれば、日付や記載名義の希望もあわせてお願いします。
```

### 3. 契約書で即答しない場合
```text
ご連絡ありがとうございます。
契約書については、内容を確認したうえで対応可否を判断したいです。
まずは、ひな形または必要条件を共有いただけますか。
確認後にお返事します。
```

## 書類作成時の注意
- 金額はココナラ上の取引内容と一致させる
- サービス名は出品名または合意済みの役務名に揃える
- 追加作業分がある場合は、合意済みの内訳だけ記載する
- 連絡先として電話番号やメールアドレスは書かない
- 住所は必要時のみ記載する

## 迷った時の判断
- 法人機能ユーザーなら、まずココナラ標準機能
- 個人相手の見積書 / 納品書なら、原則対応可
- 契約書は即答しない
- 禁止事項に触れそうなら止める
