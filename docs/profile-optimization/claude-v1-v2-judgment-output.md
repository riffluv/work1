# プロフィール v1 / v2 比較判定（Claude 出力）

作成日: 2026-04-08

---

## 1. 総合判定

**v2 を採用すべき。**

v1 に戻す理由はない。v2 は心理学調査の5理論すべてに対してv1より優位。
ただし、v2 はこのままで完成形と見てよい。微調整は不要。

## 2. 理由

### 読みやすさ: v2 が圧勝

| 観点 | v1 | v2 |
|---|---|---|
| 本文行数 | 35行 | 17行 |
| 文字数 | 約800字 | 約400字 |
| セクション数 | 7つ（自己紹介/対応内容/技術スタック/開発実績/進め方/ご注意/締め） | 4つ（導入/背景/進め方/締め） |
| 0.1秒の印象 | 「長い。情報が多い」 | 「すっきり。構造が見える」 |

v1 はセクション見出しが7つある時点で、buyer の脳は「業務マニュアルか？」と処理する。
v2 は4ブロックしかなく、各ブロックが2-4行。一目で構造が把握できる。

**認知的流暢性**: v2 が明確に勝つ。

### 安心感: v2 が勝つ（意外かもしれないが）

v1 の「技術スタック」「開発実績」「対応内容5項目」は、一見「情報が多い = 安心」に見える。
しかし、心理学調査が示す通り、**不安な状態の buyer は情報過多を「ごちゃごちゃ」と処理する**。

v2 の安心感は「進め方」3行に集約されている:
- メッセージのみ（通話なし）→ 「気軽に頼める」
- 症状と再現手順をもとに → 「手順がある＝体系的」
- 何が原因か・何を直したか → 「結果が整理されて返ってくる」

**さらに v2 は私の本命案A（進め方3行）ではなく、予備案B（進め方4行 + サービスページ導線）を採用している。**
これは正しい判断。「最終確認は依頼者様の環境でお願いしています」の1文が入っただけで、
「この人は最後まで考えてくれている」という安心感が格段に上がっている。

### 説得力: v2 が勝つ

v1 は Competence を「情報量」で示そうとしている（技術スタック4行、開発実績3項目、対応内容5項目）。
しかし、Warmth x Competence モデルが示す通り、**Competence は「行動で示す」方が強い**。

v2 の「進め方」セクションは、まさにこれをやっている:
- 「原因を特定します」「整理してお返しします」
- → 能力を直接宣言せずに、行動として有能さが伝わる

v1 の技術スタック羅列は、buyer に「で、私の問題を直せるの？」と思わせるだけ。
v2 は最初から「あなたの症状を直す」という文脈で書いているので、説得力が段違い。

### 温度感: v2 はギリギリ適温

v1 は温かいか？ → **実は温かくない。** 情報を並べているだけで、温度感がある文は最終行だけ。
v2 も温度感がある文は最終行だけ。 → **両者とも最終行に Warmth を集中させている。**

つまり、「v2 は冷たい」という懸念は、**文量が少ないこと自体を冷たさと誤認している** 可能性がある。

実際に buyer 目線で読むと:
- v1: 長い → 読み飛ばす → 温度感のある最終行まで到達しない → 結果的に冷たい
- v2: 短い → 全部読める → 最終行の「ご相談だけでも歓迎です」が確実に届く → 結果的に温かい

**Warmth が届くかどうかは、文量の多さではなく、最終行まで読まれるかどうかに依存する。**

### サービスページとの役割分担: v2 が正しい

v1 の問題は、プロフィールがサービスページの「縮小コピー」になっていること。
- 対応内容5項目 → サービスページ「こんな症状に対応」と同一
- 技術スタック → サービスページに書くべき補足情報
- 開発実績 → サービスページにすら不要かもしれない

v2 はプロフィールの役割を「サービスページへの導線」に正しく絞っている:
- 症状の具体例で「自分ごと化」を誘発 → サービスページを見に行く動機
- 「対応範囲や料金の詳細はサービスページをご覧ください」で明示的に委譲
- これは Codex が入れた1文で、非常に良い判断。

## 3. 「v2 は少し冷たいかもしれない」という懸念への答え

**冷たくない。冷たく「見える」としたら、それは短さへの不慣れ。**

根拠:
1. v2 の温度感を持つ文は最終行「ご相談だけでも歓迎です」。これは v1 にもある文。
   → 温度感の「総量」は同じ。
2. v2 が短いことで「冷たい」と感じるのは、**書き手側の感覚**。
   buyer は「短い = この人は話が早い = 頼もしい」と処理する（認知的流暢性）。
3. 心理学調査の結論: **不安な buyer は、流暢性（読みやすさ）を信頼に変換する。**
   温度感を足すために文量を増やすと、流暢性が下がり、信頼が下がる。

**結論: 「読みやすさ > 少しの温度低下」は正しい。そもそも温度は下がっていない。**

## 4. 「20万行」を残すべきか

**今の位置ならOK。**

v2 は「20万行」を、経歴文の中にインラインで1回だけ使っている:
> 個人で設計・運用しているWebアプリ（Next.js + Firebase、約20万行）で、

これは心理学的に正しい入れ方:
- セクション見出し「開発実績」の下に置く（v1）→ 列挙感 = Parvenu型
- 経歴文にさりげなく入れる（v2）→ 事実の記述 = Patrician型

Brand Prominence 理論の「ロゴが小さい方が高級」の原則に合致している。
「20万行」という数字は、括弧内にさりげなく入っているだけ。
自慢ではなく、文脈として自然に読める。

**判定: 残す。今の位置でちょうどいい。**
