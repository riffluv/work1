# Codex への依頼: ココナラプロフィール最適化

## 背景と目的

ココナラのプロフィール（自己紹介文）を最適化したい。
現在のプロフィールは「情報を詰め込みすぎて、逆に自信がなさそうに見える」という課題がある。

外部調査（#AR）で「プロフィールの深層心理」を調べた結果、
**5つの心理学理論がすべて「削ぎ落とす方向」を支持している** ことが分かった。

ただし、ココナラのアルゴリズムとの兼ね合いがあるため、
「心理学的な最適解」と「アルゴリズムの最低要件」の両方を満たす落としどころを見つけたい。

## 心理学調査の核心（5つの理論） ← Codex は必ずここを読んでから判断すること

### 理論1: 認知的流暢性（Cognitive Fluency）
**出典: Princeton大学（Alter & Oppenheimer）、Global Council for Behavioral Science（2026）**

人間は「処理しやすい情報」を「正しい情報」だと無意識に感じる。

- 読みやすいフォントで書かれた文章 → より「真実だ」と判断された
- 発音しやすい株式銘柄 → IPO直後にパフォーマンスが良い
- 明瞭なフォントの哲学的議論 → より「説得力がある」と判断された
- 全部、内容は同じ。**見た目の「処理のしやすさ」だけが違う**

**重要な発見**: 信頼の流暢性効果が最も強くなる条件:
1. 相手が**不安を感じている時**（← buyer はバグで困っている = まさにこの状態）
2. **時間がない時**（← ココナラで比較検討中）
3. **早く答えが欲しい時**（← 本番が壊れている）

→ バグで困ってココナラを見ている buyer は、流暢性効果を最も強く受ける。
→ **シンプルなプロフィールが最大の武器になる**。

### 理論2: Brand Prominence / ステータスシグナリング
**出典: Han, Nunes & Drèze（USC, Journal of Marketing, 2010）**

実験で確認された事実:
- **偽ブランド品は、本物よりロゴが大きい**（Brand Prominence が高い）
- **本物の高級品でも、高いものほどロゴが小さい**
- **価格と Brand Prominence は逆相関する**

4つの消費者タイプ:
- **Patrician（真の富裕層）**: ロゴの小さい高級品を好む。「分かる人だけ分かればいい」
- **Parvenu（成り上がり）**: ロゴの大きい高級品を好む。「みんなに見せたい」

プロフィールへの翻訳:
- できることを全部書く = Parvenu型 = 「この人は認めてほしいんだな」
- 削ぎ落とす = Patrician型 = 「この人はアピールする必要がないくらい自信がある」

### 理論3: Warmth x Competence モデル
**出典: Susan Fiske, Amy Cuddy, Peter Glick（Princeton, 2007）**

人間は他者を**「温かいか（Warmth）」と「有能か（Competence）」の2軸**で瞬時に判断する。

- **理想**: Warmth ↑ + Competence ↑ = 「尊敬+信頼」
- **問題**: Competence を過剰にアピールすると → Warmth が相対的に下がる
  → 「有能かもしれないが、冷たそう」→ 嫉妬や不信を招く

**プロフィールへの適用**:
- スキルを並べすぎる = Competence 過剰 → Warmth が下がる → 「頼みたい」が弱くなる
- **Competence は「行動で示す」**（例: 「48時間以内に一次報告」← 能力を直接言わずに示す）
- **Warmth は「1文だけ」足す**（例: 「困っている時に頼れる場所を作りたくて出品しました」）

### 理論4: Thin-Slicing（瞬間判断）
**出典: Dutable（2026）**

- 0.05秒: 視覚的な印象を形成（アイコン、空白の量、**文字量**）
- 0.1秒: 「この人は信頼できるか」の暫定判断が完了
- 2-3秒: 暫定判断を「確認する」方向で情報を読む（確認バイアス）

→ テキストの「量」が多いだけで、0.05秒で「ごちゃごちゃ」と判断される
→ その後のテキストは確認バイアスで読まれる（つまり挽回が難しい）

**Halo Effect**: 第一印象が良い → 全てが良く見える。逆も然り。

### 理論5: Quiet Luxury（静かな贅沢）
**出典: Social Life Magazine, Berkeley Technology Law Journal（2024-2026）**

- 2026年: **「説明しない」がステータスシグナル**
- 控えめであること自体が最大の自信の表れ
- 「分かる人だけ分かればいい」= 真の強者の態度

プロフィールに適用すると:
- ❌ 「10年以上の経験を持つフルスタックエンジニアが、高品質なコードレビューと不具合修正を提供します」
- ✅ 「Next.js / Stripe の不具合を直します。」
- → 後者の方が「強者感」がある

### 5つの理論の結論
```
すべてが「削ぎ落とす」方向を指している:

  1. 認知的流暢性: 短い方が「正しい」と感じる
  2. Brand Prominence: 書くほど「必死」に見える
  3. Warmth x Competence: スキル羅列は Warmth を殺す
  4. Thin-Slicing: 0.1秒の文字量の印象が全て
  5. Quiet Luxury: 説明しないことが最大の自信

ただし:
  ココナラのアルゴリズム: 空欄はスコアが下がる
  → 「フィールドは全部埋める。ただし中身は最小限にする」
  → これが両方を満たす唯一の解
```

## 分析結果: 何を残し、何を削るか

### 即削除（サービスページに既にある or buyer が読まない）
- **技術スタック（4行）** → サービスページに委譲。プロフィールに書くと Competence 過剰
- **開発実績（3行）** → buyer は「あなたの作品」に興味がない。「私の問題を直せるか」に興味がある
- **対応内容の詳細5項目** → サービスページと完全に重複。症状ベースで2行に圧縮可能

### 圧縮
- **経歴（3行 → 1文）** → 「20万行のWebアプリを個人で運用」は強いシグナルだが、1文で十分
- **進め方（4行 → 3行）** → 行動で Competence を示す重要パート。残すが微圧縮

### そのまま残す
- **ひとことアピール** → 完成度が高い。触らない
- **進め方の核心** → 行動で有能さを示す。心理学的に正しい
- **ご注意（制限の明示）** → 専門性の証明。Quiet Luxury の原則に合致
- **最終行（Warmth の1文）** → 人間味。これが trust の最後のピース
