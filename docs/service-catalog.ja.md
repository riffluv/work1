# サービスカタログ（運用台帳）

更新日: 2026-04-02

## 目的
- 複数サービスを運用しても、サービス定義と案件の紐付けを崩さない。
- セッションが切れても、次のCodexがすぐに「どのサービスで進めるか」を判別できるようにする。

## 運用ルール（固定）
- 新しいサービスを追加したら、この台帳に `Service ID` を先に追加する。
- 新規案件の `README.md` には必ず `Service ID` を記入する。
- 価格や範囲を変更した場合は、台帳と該当正本を同時更新する。
- 価格・追加料金・公開状態の機械判定は、この台帳ではなく `service-registry.yaml` と各 `facts_file` を正本にする。
- `bugfix-15000` は「一次ソース更新 -> docs同期」の順で更新する。
- 「運用中 / 将来案 / 廃止」を明示し、混在させない。
- 外向けに案内してよいのは、ステータスが `運用中` のサービスだけとする。
- `handoff-25000` は `登録済み・未公開` の間、内部準備用の参照にとどめ、外向け返信でサービス名・価格・購入導線を出さない。

## 引き継ぎルール
- セッション開始時は `次セッション用_起動プロンプト.txt` → `docs/next-codex-prompt.txt` → `os/core/boot.md` の順で読む。
- 案件対応時は、`Service ID` から参照すべき正本を決めてから返信・実装する。
