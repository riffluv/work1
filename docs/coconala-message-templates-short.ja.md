# ココナラ短文テンプレ（実運用）

## 使い方
- `[]` を置換して送信
- 1通は「結論→次アクション→期限」の順
- 迷ったら短くして先に返信（48時間無連絡を避ける）

## 時刻未確定時の定型
- まず、2時間以内（JST）を目標に方向性をお返しします。
- 他に必要なことがあれば、こちらからお聞きします。

## 購入後フェーズの時刻未確定定型
- まず、2時間以内（JST）を目標に次の確認結果をお返しします。
- そのあと、48時間以内を目標に一次結果をご案内します。

## 末尾ローテーション定型（2時間/48時間）
- この返信から2時間以内（JST）を目標に方向性をお返しします。  
そのあと、48時間以内を目標に一次結果をご案内します。
- 2時間以内を目標に、まず次の確認結果をお返しします。  
続けて、48時間以内を目標に一次結果をご案内します。
- まず2時間以内（JST）を目標に判定の見立てを共有します。  
その後、48時間以内を目標に一次結果をご案内します。

## 完了確認フェーズの末尾定型（二段構成不要）
- ご返信いただければ、すぐに次の進行を確定します。
- A/Bどちらかご返信いただければ、このまま進行を確定します。

## 9. 自分用チェック（送信前）
- 依頼範囲外を無料で混ぜていないか
- 次アクションと期限が書いてあるか
- 外部連絡や秘密情報要求になっていないか
- 追加質問の前に受領確認の1文（「確認しました」など）を入れているか
- 文章が長すぎないか（要点先頭）
- 1つの質問項目に依頼を詰め込みすぎていないか（1項目1情報）
- 非エンジニア向けで「ログ」単語を単独要求していないか（画面スクショ優先）
- 「対応範囲外です」で終わらず、代替案を添えているか
- 「見込み + 別原因は事前相談」の文が連続で同一になっていないか
- 見積り初回で料金を並べすぎていないか（基本料金 + 追加は事前相談まで）
- セキュリティ注記が「禁止 -> 代替 -> 事後対処」の順になっているか
- 判定前に追加料金の具体金額（5,000円 / 15,000円）を出していないか
- 購入後フェーズの返信に、次回時刻（本日中 / [時刻]まで / 2時間以内）が入っているか
- 責任/保証の回答が「保証範囲 -> ただし -> 手順」の3段構成になっているか
- 調査/判定フェーズで `2時間以内` と `48時間以内` の二段構成が両方入っているか
- 完了確認フェーズで二段構成を機械適用していないか
- `2時間以内` が「受領確認 or 一次報告」の意味で使われ、一次結果の時刻が別で明記されているか
- 追加質問の末尾に「分からなければスクショ1枚でOK」のフォールバック文を入れたか
- `対応していません。` で終わらず、柔らかい表現 + 代替案になっているか
- `補償` / `損害補償` を先に使わず、`責任範囲` / `対応方針` で書いているか
- 書き出しと末尾2行が同じ表現で連続しすぎていないか（3件連続を避ける）
- `[ ]` / `{ }` のプレースホルダーが残っていないか
- 購入者の適切な行動（バックアップ取得・情報整理など）を拾って1文で認めているか

## 20. 責任/保証に関する質問への回答
[お名前]様  
率直にご不安を共有いただきありがとうございます。  
本サービスは、原因特定と修正内容の作成・検証手順の提示までを担当し、最終適用と最終確認は依頼者様環境でお願いしています。

別原因の不具合が発生した場合は、切り分けのうえ追加対応として事前にご相談します。  
リスクを下げるため、以下の手順を推奨します。  
1) 適用前バックアップ  
2) 最小単位での反映  
3) 反映直後の確認手順の実施
