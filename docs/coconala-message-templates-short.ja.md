# ココナラ短文テンプレ（実運用）

## 使い方
- `[]` を置換して送信
- 1通は「結論→次アクション→期限」の順
- 迷ったら短くして先に返信（48時間無連絡を避ける）

1) 症状（いつ、どこで、何をしたとき）  
2) エラー文やログ（マスク済み）  
3) 再現手順（操作の順番を1,2,3で）  
本日[時刻]までに一次報告を返します。

## 2. 情報不足の追加確認
追加で2点だけ確認させてください。  
1) [確認事項1]  
2) [確認事項2]  
[日付/時刻]までにご返信いただけると、切り分けを進められます。

- 判明: [分かったこと]

- 予定: [次回報告時刻]

追加分は [有料オプション/おひねり] で [金額]円、納期は [日付] で対応可能です。進め方をご指定ください。

1) [変更点1]  
2) [変更点2]

## 6. 正式な納品メッセージ
正式な納品をお送りします。  
- 納品物: [修正済みZIP（標準）/patch（任意）/レポート]

ご確認のうえ、必要があれば差し戻しでご連絡ください。

## 7. 返信が止まったとき

A) このまま進めてOK  
B) 修正点あり（箇条書きで共有）  
C) いったん保留（再開目安だけ共有）

## 8. 評価返信
この度はご依頼ありがとうございました。  
要件共有が明確で、スムーズに進行できました。  
また不具合調査や改善の際はお気軽にご相談ください。

## 9. 自分用チェック（送信前）
- 依頼範囲外を無料で混ぜていないか
- 次アクションと期限が書いてあるか
- 外部連絡や秘密情報要求になっていないか
- 文章が長すぎないか（要点先頭）

## 10. 見積り相談の初回返信（受注前）
### A) 非エンジニア向け（標準）
[お名前]様  
ご相談ありがとうございます。まず状況を確認し、対応できるか判断します。  
まず次の3点を教えてください。  
1) どうなってほしいか（本来の動き）  
2) 実際に何が起きているか  
3) どこで起きるか（本番 / プレビュー / ローカル）  
例: 本番で購入ボタン押下後に500エラー。決済は成功するがDBに反映されない。  
分かる範囲で、次もお願いします。  
- いつ頃から発生したか  
- 不具合が起きるまでの操作手順（1,2,3の順番）  
- エラーメッセージや画面スクショ（分かれば利用技術: Next.js / Stripe / Vercel など）  
※.envの値など秘密情報は送らないでください。  
いただければ、[日付/時刻]までに「不具合1件の範囲判定」と見積り案を返します。

### B) エンジニア向け（要点重視）
[お名前]様  
ご相談ありがとうございます。対応可能か確認しますので、まず次の3点をお願いします。  
1) 期待挙動（本来の動き）  
2) 実挙動（実際の不具合）  
3) 発生環境（本番 / プレビュー / ローカル）  
例: 本番でcheckout後に500。決済成功だがDB更新失敗。  
分かる範囲で、次も共有ください。  
- 発生開始時期  
- 再現手順（1,2,3）  
- エラーメッセージ / スクショ（分かれば技術構成: Next.js / Stripe / Vercel）  
※.env値やAPIキー値は不要です（キー名のみ共有可）。  
いただければ、[日付/時刻]までに「不具合1件の範囲判定」と見積り案を返します。

## 11. 1件判定（基本15,000円）の提示
内容を確認しました。今回のご相談は、同一原因に紐づく1フロー/1エンドポイントとして判定できます。  
基本料金15,000円で、原因切り分けから修正済みファイル（ZIP・標準）または修正差分（patch・任意）の共有まで対応します。  
最終確認は依頼者様環境でお願いし、確認結果をいただければ追加調整まで対応します。  
この内容で見積りを送ってよければ「このまま進めてOK」とご返信ください。

## 12. 複数件判定（追加料金あり）の提示
内容を確認しました。今回のご相談は、原因または対象フローが分かれるため、2件対応の扱いになります。  
- 1件目: 基本料金 15,000円  
- 2件目: 追加修正1件 15,000円  
まずは1件目のみ着手し、2件目は結果を見て追加判断する進め方も可能です。  
希望の進め方（A:2件同時 / B:1件目優先）をご指定ください。

## 13. 調査延長オプション（30分）の提示
現状、原因候補は[候補]まで絞れていますが、確定には追加調査が必要です。  
追加調査30分（5,000円）で、ログ確認と再現検証を進めて確定報告します。  
進める場合は見積りを更新しますので、「追加調査で進行OK」とご返信ください。

## 14. 見積り送信前チェック（自分用）
- 依頼経路を確認したか（サービス経由 / プロフィール経由 / メッセージ経由）
- 初回見積りは「必須3項目 + 任意3項目」で聞けているか
- 発生環境の表記を「本番 / プレビュー / ローカル」で統一したか
- 提案時に「出品サービスに紐付ける」を設定したか
- 不具合1件の定義（同一原因 / 1フロー / 1エンドポイント）で説明したか
- 範囲外（通話、直接 push、本番デプロイ、新機能開発）を明記したか
- 相手が返しやすい選択肢（A/Bなど）を入れたか
