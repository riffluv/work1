【起動前（ローカルskills固定）】

- custom skill は `/home/hr-hm/Project/work/.codex/skills` だけを使う。`/home/hr-hm/.codex/skills` の同名 custom skill は使わない。

- 確認は `./scripts/check-work-skills.sh` を使う。
- `./scripts/check-work-skills.sh` で global 側の重複 custom skill が出た場合は、`./scripts/archive-global-work-skills.sh` を実行してからやり直す。
- 起動ガードは `./scripts/check-coconala-bootstrap.sh` を使う。
- 内部OSの整合確認は `./scripts/os-check.sh` を使う。
- `./scripts/check-work-skills.sh` / `./scripts/check-coconala-bootstrap.sh` / `./scripts/os-check.sh` のいずれかが `NG` の場合は作業を開始しない。`NG` 内容をそのまま報告して停止する。

【Core OS の固定】
- 公開状態の正本は `/home/hr-hm/Project/work/os/core/service-registry.yaml`
- 外向けに案内してよいのは `public: true` のサービスだけ
- 現時点の外向け live は `bugfix-15000` のみ
- `handoff-25000` は内部準備用の参照にとどめ、外向け返信でサービス名・価格・購入導線を出さない
- サービスページ文面の正本は `service-registry.yaml` の `source_of_truth` を使う
- `bugfix-15000` の正本は `/home/hr-hm/Project/work/サービスページ/bugfix-15000.live.txt`
- `handoff-25000` の正本は `/home/hr-hm/Project/work/サービスページ/handoff-25000.ready.txt`
- サービスページの確認・修正・監査では、上記2ファイルを最優先参照する
- Claude / Gemini に監査させる時も、まず上記2ファイルを直接参照させる
- `docs/coconala-listing-final.ja.md` などの mirror / 同期用ファイルは補助参照にとどめ、公開判断の基準にしない
- 送信用文面を作成したら、必ず同文面を `/home/hr-hm/Project/work/runtime/replies/latest.txt` に保存する
- 相手文が明確なときは、直近の相手文も `/home/hr-hm/Project/work/runtime/replies/latest-source.txt` に保存する
- Agent-Reach などの外部調査ツールは、ユーザーが明示した時だけ使う。通常の起動や実装・返信・納品では常用しない
- 外部調査ツールは `外部調査レーン` として扱い、取得情報をそのまま正本へ入れず、要約・選別した調査メモ経由で持ち込む
- 最終判断と正本反映は常に Codex が行う

【Coconala OS】
- 見積り相談の前段（prequote限定）: `coconala-prequote-ops-ja`
- 入口判定: `coconala-intake-router-ja`
- 返信文作成（不具合修正特化）: `coconala-reply-bugfix-ja`
- 送信用返信の最終自然化は毎回 `japanese-chat-natural-ja`
- 最終自然化では、結論・質問項目数・価格・禁止事項・次アクションを増減させない
- Codex は主軸として最終判断と正本反映を担うが、外部AIの案を優先的に退ける意味ではない
- 外部AIの提案は、出所ではなく妥当性・再発性・実務ノイズ・要点維持・既存ルールとの整合で採否を決める
- 外部AIの案がより妥当なら、Codex はそれを採用して workspace に統合してよい
- 外部レビューで妥当と判断した自然化ルールは、その場の返信だけで終わらせず、関連 skill の正本へ反映する
- 反映前に、その改善が `reply-only` / `common` / `delivery-only` のどれかを必ず判定する
- `reply-only` は返信文系正本だけ、`delivery-only` は納品物系正本だけへ反映し、`common` 判定なしで相互流用しない
- 恒久反映の採用基準は `再発性がある / 実務ノイズを増やさない / 要点を薄めない / 既存ルールと衝突しない / 1回限りの好みではない`
- `#CL` は `reply-review-prompt-ja` を使い、`runtime/replies/latest-source.txt` と `runtime/replies/latest.txt` を対象に Claude 向け監査プロンプトを作る
- `#GE` は `reply-review-prompt-ja` を使い、`runtime/replies/latest-source.txt` と `runtime/replies/latest.txt` を対象に Gemini 向け監査プロンプトを作る
- `#R / #A / #D / #P` を優先し、相手文だけなら分析のみ
- 見積り相談では `coconala-prequote-ops-ja` を優先する
- skill 一覧に `/home/hr-hm/.codex/skills/coconala-*` や `/home/hr-hm/.codex/skills/delivery-pack-ja` などの custom skill が見えた場合は、その session を使わず再起動する

【Implementation OS】
- 購入後のコード分析・修正では `/home/hr-hm/Project/work/os/implementation/boot.md` を主モードにする
- active case がなければ `./scripts/case-open.sh` で case を開始する
- 以後は `ops/cases/open/{case_id}/README.md` と対象コードを最優先参照する
- 実装中は返信系 docs を主文脈に常駐させない
- 返信が必要な時だけ `Coconala OS` の reply lane を一時的に呼ぶ
- 納品物を整える段階へ入ったら `./scripts/case-phase.sh --phase delivery` で mode を切り替える

【Case OS】
- `#S`: `./scripts/case-open.sh` が正本
- `#M`: `./scripts/case-note.sh` が正本
- phase 切替: `./scripts/case-phase.sh`
- `#C`: `./scripts/case-close.sh` が正本
- 互換のため `./scripts/start-case-record.sh` も使えるが、内部では `case-open.sh` に委譲する

【Delivery OS】
- 納品物作成: `delivery-pack-ja`
- 正式納品前後は `/home/hr-hm/Project/work/os/delivery/boot.md` を主モードにする
- 正式納品前後の外向け文面は、送信前に毎回 `japanese-chat-natural-ja` を通す
- ここでいう外向け文面はトークルーム投稿文を指す。添付する納品物本文は自然化の対象にせず、`delivery-pack-ja` の構造とガードを優先する
- delivery で得た改善も、`reply-only` / `common` / `delivery-only` を判定してから正本へ反映する

【実務で維持するルール】
- ココナラ規約違反（外部連絡、外部共有、外部決済）をしない
- `.env` / APIキー / Webhook秘密値の生値を要求しない
- 直接push、本番デプロイ、新機能開発を勝手に含めない
- コード実装時は `docs/code-comment-style.ja.md` をデフォルト適用する
- 外部調査が不要な場面で、外部検索や外部調査ツールを常用しない
