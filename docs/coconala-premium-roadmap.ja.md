# 不具合修正プレミアム 出品ロードマップ

更新日: 2026-02-14

## 目的
- 現行サービス（15,000円）で実績を作った後、上位サービス「不具合修正プレミアム」を追加出品する。
- 単価を上げつつ、現行サービスとの役割分担を明確にして運用を安定させる。

## 現行サービス（入口）
- 基本料金: 15,000円
- 内容: 不具合1件（同一原因 / 1フロー or 1エンドポイント）
- 役割: 相談の入口、初回受注、レビュー蓄積

## プレミアムサービス（将来追加）
- 想定価格: 30,000〜60,000円（初期推奨 39,800円〜49,800円）
- 想定内容:
1. 原因特定（複数仮説の整理）
2. 修正2件（同一案件内）
3. 回帰チェック項目の提示
4. 再発防止メモ（運用/監視/チェックポイント）
- 標準範囲外: 通話、直接push、本番デプロイ、新機能開発

## 出品条件（Go条件）
以下を満たしたら追加出品する。

1. 現行サービスで1〜2件を納品完了
2. 可能なら★5評価を1件以上取得
3. 工数感が読めている（修正1件あたりの目安時間）
4. テンプレが固まっている
   - 初回返信
   - 追加料金案内
   - 納品文
   - スコープ変更通知

## 導入ステップ
1. 現行サービスを運用し、案件ログを残す
2. 案件ごとに「何に時間がかかったか」を記録
3. プレミアム用の本文・FAQ・価格を作成
4. 現行サービスは残したまま、別サービスとして公開
5. 見積り時に条件が合う案件だけプレミアムへ誘導

## 役割分担（2サービス併用）
- 現行（15,000円）: 小〜中規模、単発、入口相談
- プレミアム（30,000〜60,000円）: 複数修正、再発防止まで必要な案件

## 見積り時の判定ルール（簡易）
- 原因が単一 + 修正1件で完結: 現行
- 原因が複合 + 修正2件以上 + 検証設計が必要: プレミアム

## KPI（公開後30日）
1. プレミアム見積り相談数
2. 見積り→購入率
3. 納品完了率
4. 平均単価
5. 低評価件数（0維持）

## 運用上の注意
- 現行サービスを消さない（入口を失わないため）
- 価格だけ上げず、納品物の違いを明確化する
- スコープ定義を必ず本文とFAQの両方に書く
- 無理な案件は受けず、スコープ変更通知で調整する
