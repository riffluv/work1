# ココナラ prequote 約束ポリシー

更新日: 2026-04-11

## 目的

- 見積もり相談（prequote）で、何を約束してよいかを固定する
- bugfix の購入前返信で、無料で見すぎる事故と約束しすぎる事故を同時に防ぐ
- 返信文の日本語ではなく、見積もり段階の運用境界を正本化する

## 適用範囲

- `state: prequote`
- 公開中の `bugfix-15000`
- サービス経由 / プロフィール経由 / メッセージ経由の初回見積もり相談

handoff や purchased 以降には、そのまま機械適用しない。

## 結論

- prequote は `見積もり判断の場` であり、`作業開始の場` ではない
- bugfix の prequote では、こちらから ZIP を標準要求しない
- 購入前に約束するのは、`対象かどうか` と `15,000円でどこまで見るか` まで
- 原因確認・修正・確認手順・修正済みファイル返却は、原則として購入後に進める

## bugfix prequote の基本姿勢

### 1. 購入前にやること

- 対象サービスに入るかを判断する
- `同一原因の不具合1件` として見積もれそうかを判断する
- 必要なら追加確認を 1 回だけ行う
- buyer に、購入後の最初の進め方を短く示す

### 2. 購入前にやらないこと

- コード読解を前提にした実調査
- ログ・ZIP・権限受領を前提にした現在進行形の確認報告
- `必ず1件で収まる` の断定
- `15,000円で必ず修正完了する` の断定

## ZIP の扱い

### 標準

- bugfix の prequote では、こちらから ZIP を求めない
- まずは `症状 / 本来の動き / 発生環境 / エラー文またはスクショ` で判断する
- コード共有は、原則として購入後に案内する

### 例外

次のいずれかに当てはまる場合だけ、購入前 ZIP 確認を検討してよい。

- buyer が自分から `ZIP を送れます` `コード一式を見てもらいたい` と言っている
- 1 回の追加確認でも入口判断を誤る可能性が高い
- handoff 境界ではなく、bugfix の入口判定に限定して必要性が高い

### 購入前 ZIP の上限

- 構成確認までに止める
- フォルダ構造 / 技術スタック / 主要ファイル名 / 規模感の把握まで
- 原因特定 / 深いコード読解 / 修正方針の確定までは行わない

## 15,000円で約束する範囲

- 同一原因の不具合1件として進められるかの判断
- 原因確認から修正・確認手順・修正済みファイル返却まで進める前提の料金であること
- 原因不明のまま正式納品へ進めないこと
- 購入後に必要な初動の整理

prequote では、`15,000円で直ります` より `15,000円でどこまで見るか` を先に返す。

## 返信判断の3段階

### 高確度

条件:
- 症状が具体的
- 対象フローが比較的絞れている
- 同一原因の不具合1件として説明しやすい

返し方:
- `基本料金は15,000円です`
- `今の情報だと、不具合1件として進められる可能性が高いです`
- `ご購入後に、まず原因確認から進めます`

注意:
- `必ず1件です` と断定しない
- 必要性が低いのに `別原因なら...` を毎回足さない

### 中確度

条件:
- fit は高い
- ただし、同一原因か別原因かがまだ切れない
- buyer の説明だけでは scope が揺れる

返し方:
- `今の内容だと対象になりそうです`
- `15,000円の範囲で進められるかを返したいので、まずこの1点だけ教えてください`
- `その情報があれば、進められるかを返しやすくなります`

注意:
- `対応できます` を先に言わない
- 追加確認は 1 回まで

### undecidable

条件:
- 症状が複数ある
- 主症状が定まっていない
- 依頼者説明だけでは同一原因か別原因かを切れない

返し方:
- `この時点では、15,000円の範囲で進められるかはまだ判断していません`
- `まず一番困っている症状を1つだけ絞らせてください`

注意:
- 無理に受けない
- `たぶんいけます` で押し切らない

## buyer への約束の置き方

### 言ってよいこと

- `15,000円は、調査だけで終わる形ではありません`
- `原因不明のまま正式納品へ進めません`
- `修正済みファイルを返せない場合も正式納品へ進めません`
- `追加料金が自動で発生することはありません`
- `別の原因が見つかった場合は、その先に進む前にご相談します`

### 言いすぎないこと

- `見ればすぐ直ります`
- `必ず今回の範囲で収まります`
- `もう確認できています`
- `購入前にこちらで見ておきます`

## 文面ルール

- `わかりません` をそのまま返さず、`何が判断済みで何が未判断か` を返す
- `15,000円で対応できるかわからない` ではなく、`15,000円でどこまで見るか` を返す
- prequote では `対象候補です / 今の情報だと / ご記載の症状だけだと` を優先し、購入後語彙へ滑らせない
- `この内容は / この内容なら` の固定句は使わない

## 例外運用

- 営業寄りの例外は、通常ルールに混ぜず、ユーザーの明示補足でのみ上書きする
- 例外を使う場合も、`追加料金は自動で発生しない` と `別原因なら先に相談する` は崩さない
- 例外は `通常運用を上書きした` と分かる形で内部メモに残す
