# 返信システム マネタイズ案メモ

## 前提

- 返信システムの品質が 1.5 → 4.5 に改善済み
- 市場には「ChatGPT にプロンプト1枚渡す」レベルしかない
- 自分が持っているのは「プロンプト」ではなく「設計思想 + 仕組み」
- template 回帰の問題に気づいている人が市場にいない ← 最大の差別化

## 売れるもの一覧

### 1. 知識を売る（note 有料記事シリーズ）

```
記事1: 構築記録（無料 or 序盤無料）
  → 集客用。「AI 返信を 1.5 → 4.5 にした話」
  → note の SEO + SNS 拡散で流入

記事2: テンプレート回帰の罠と対策（有料 1,000〜3,000円）
  → 「ChatGPT に返信を書かせるとなぜ全部同じになるのか」
  → 多くの人がハマっている問題を言語化

記事3: 3つの AI を使い分ける方法（有料 1,000〜3,000円）
  → Codex / Claude / Gemini の役割分担ノウハウ

記事4: regression suite の作り方（有料 1,000〜3,000円）
  → 品質を数字で測る方法
```

### 2. 設計キットを売る（BOOTH / note）

```
商品名: 「ココナラ受託 返信AI設計キット」

内容:
  1. AGENTS.md テンプレート（サービス定義の正本テンプレ）
  2. service.yaml テンプレート（capability map + diagnostic patterns）
  3. response_decision_plan の設計ガイド
  4. regression テストケース 10件（カスタマイズ用）
  5. Claude/Gemini 監査プロンプト（コピペで使える）
  6. 導入ガイド（Codex / Claude Code でこう使ってください）

価格帯:
  BOOTH:   5,000〜10,000円（ZIP パッケージ）
  note:    3,000〜5,000円（解説記事 + ダウンロード）
```

### 3. ココナラで新サービス出品

```
サービス名: 「あなたのサービス用 AI 返信設計を作ります」

内容:
  - 依頼者のサービス内容をヒアリング
  - capability map / diagnostic patterns を設計
  - AGENTS.md をカスタマイズ
  - テストケース 5件を作成
  - 導入サポート

価格帯: 15,000〜25,000円
対象: ココナラ出品者 / フリーランス
```

### 4. 監査プロンプト単品販売

```
商品名: 「AI 返信の品質チェックプロンプト」

内容:
  - Claude / Gemini に渡すだけで返信文を監査してくれるプロンプト
  - 「主質問に答えているか」「bot感がないか」等のチェック項目付き
  - 使い方ガイド

価格帯: 1,000〜3,000円（BOOTH / note）
→ 低価格で買いやすい入口商品
```

## 差別化ポイント

```
他のプロンプト商品:
  × 「プロンプト1枚で返信が書ける」
  × 品質 1.5 のまま
  × template 回帰に気づかない
  × 使い続けると全部同じ返信になる

自分の商品:
  ○ 設計思想ごと渡す（OS を売る）
  ○ 品質 4.5 に近づける仕組み
  ○ regression suite で品質が落ちない
  ○ 「なぜテンプレに戻るのか」の構造理解を含む
  ○ 数字で品質を証明できる（1.5 → 4.5）
```

## 販売チャネル比較

| チャネル | 相性 | メリット | デメリット |
|---|---|---|---|
| note 有料記事 | ◎ | 解説系に強い / SEO で流入 | ファイル添付が弱い |
| BOOTH | ○ | デジタルデータ販売特化 / 技術系ユーザー多い | 集客は自力 |
| ココナラ | ○ | 既存の出品者にリーチ / サービス形式 | レッドオーシャン |
| Gumroad | △ | 海外向け / テンプレ販売に強い | 日本語ユーザー少ない |

## 推奨ロードマップ

```
Phase 1（今すぐ）:
  → note 記事を完成させる（無料 or 安価）
  → 集客の入口を作る

Phase 2（返信システム安定後）:
  → BOOTH で設計キットを販売
  → note 有料記事シリーズ開始

Phase 3（実績が溜まったら）:
  → ココナラで「返信AI設計」サービス出品
  → 既存の bugfix-15000 と並べて横展開
```

## 注意事項

- 自分の workspace の秘密情報（.env / API キー）は含めない
- AGENTS.md のココナラ固有ルールは汎用化してから売る
- テストケースは実案件そのままではなく、汎用化する
- 「これを買えば全自動」ではなく「設計を理解するキット」として売る
  → 過度な期待を持たせない → クレーム防止
