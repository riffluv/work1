# 動作確認の手順と結果

## ご確認の手順
1. テスト環境で対象プランの決済を1件実行してください
2. Stripe ダッシュボードの `開発者 -> イベント` を開き、対象イベントが成功になっているか確認してください
3. アプリ側で対象ユーザーの会員ステータスが有料へ切り替わっているか確認してください
4. サーバーログに署名検証エラーが出ていないか確認してください

## こちらで確認した内容
- 修正差分上、署名検証前に `request.json()` を読む処理がなくなっていることを確認
- raw body を `request.text()` で取得する形に変わっていることを確認
- 署名検証失敗時にログが残るため、再発時の切り分けがしやすい状態になっていることを確認
- `event.id` を使った重複排除の前提が維持されていることを確認

## 依頼者様の環境で見てほしいポイント
- テスト決済後に有料プランへ正しく反映されるか
- 同じ決済で二重反映が起きないか
- Stripe 側のイベント配信履歴で失敗が残っていないか

## もし問題が残っている場合
次のいずれかを共有いただければ、同一原因の範囲で追加確認できます。

- Stripe イベント詳細画面のスクショ
- 失敗時のサーバーログ
- 決済後の会員ステータス表示画面のスクショ

.env の値は送らず、キー名のみで大丈夫です。
