# 00_結論と確認方法.md

## 1. 今回の対象
- 対象不具合: 決済成功後に会員状態が有料へ更新されない
- 対象環境: Next.js App Router / Stripe / Vercel / 本番環境
- 今回の対応範囲: Stripe Webhook 受信から会員状態更新まで
- 今回は見ていない範囲: Billing Portal 側の画面表示と管理画面の文言

## 2. 結論
- 状態: 今回の不具合は解消しました
- 主原因: Webhook 署名検証前に request body を JSON として読み出しており、本番だけ署名検証に失敗していました
- 依頼者さまにお願いしたい確認: テスト決済を1件実行し、決済後に対象会員のステータスが有料へ変わるかご確認ください

## 3. 確認手順
1. テスト決済を1件実行する
2. Stripe ダッシュボードで `checkout.session.completed` が成功していることを確認する
3. アプリ側で対象会員のステータスが有料へ更新されていることを確認する

## 4. 原因
Webhook 署名検証は raw body が必要ですが、修正前は `request.json()` を先に通していました。  
ローカルでは通っていても、本番では署名の元データが変わり、`checkout.session.completed` の検証で失敗していました。

## 5. 修正内容
- 対象: `app/api/stripe/webhook/route.ts`
- 変更: `request.text()` で raw body を受け、署名不正は `400`、一時障害は `500` で返す形に整理
- 修正方針: 署名検証と会員状態更新を分け、再送時も同一 `event.id` を二重適用しないように修正

## 6. 変更ファイル
- `app/api/stripe/webhook/route.ts`: raw body で署名検証し、エラー時のHTTP方針を整理
- `src/lib/billing/applySubscriptionUpdate.ts`: `event.id` で重複更新を防ぐ処理を追加

## 7. こちらで確認した結果
- Webhook 受信から会員状態更新までの流れをコード上で追える状態になりました
- 同一 `event.id` の再送で会員状態が二重更新されない形になりました

## 8. 未確認 / 依頼者環境でのみ確認できる点
- Vercel 本番環境の実イベントを使った最終確認は、依頼者さま側でお願いします
- 本番環境では、対象会員のステータス更新までご確認ください
