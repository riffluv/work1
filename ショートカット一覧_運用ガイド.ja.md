# ショートカット一覧（運用早見表）

## まずはこれだけ（最小運用セット）
- `#S`
- `#C`
- `#$0`
- `#R 受領返信`
- `#P`
- `#CL`
- `#GE`
- `#R 方向性返信`
- `#R 正式納品案内`
- `#R クローズお礼`

## 先に返す定形文
- 分析や文面調整に少し時間が欲しい時は、まずこれを返す
- `承知しました。確認して、改めてお返しします。`

## 対象外の懸念を見つけた時
- 先に追加料金の話をしない
- まずは静かに共有する
- 定形:
  - `今回の対象を確認する中で、別の箇所にも少し気になる点がありました。今回の対応とは別ですが、参考までに共有します。`

## 今のサービス外だが技術的には受けられる時
- 先に `できません` で切らない
- まずは `今のサービスとは別の対応` と伝える
- 金額は先に出さない
- 定形:
  - `今のサービスとは別の対応になりますが、技術的にはできます。必要であればお見積りします。`

## 使い分け解説表（迷いやすい組み合わせ）
| 迷う組み合わせ | 使い分け基準 | 判断ポイント |
|---|---|---|
| `#S` / `#$0` | `#S` は内部記録開始。`#$0` は購入者向け初回返信。 | 記録か返信か |
| `#M` / `#R 方向性返信` | `#M` は内部メモ保存。`#R 方向性返信` は購入者向け中間報告。 | 内部記録か外向け返信か |
| `#C` / `#R クローズお礼` | `#C` は内部の案件保存。`#R クローズお礼` は購入者向け締めの文面。 | 保存か返信か |
| `#$0` / `#R 受領返信` | `#$0` は購入直後の初回。`#R 受領返信` は資料が届いた直後。 | 初回開始か、資料受領後か |
| `#P` / `#R 方向性返信` | `#P` は調査中報告。`#R 方向性返信` は原因候補を示せる段階。 | 見立てが出せるか |
| `#R 納品前確認` / `#R 正式納品案内` | `#R 納品前確認` はボタン前。`#R 正式納品案内` はボタン後。 | 納品ボタンを押したか |
| `#$3` / `#R 承諾前フォロー` | `#$3` は見積り提案後の未返信。`#R 承諾前フォロー` は正式納品後の未返信。 | 購入前か納品後か |
| `#R 承諾前フォロー` / `#R クローズお礼` | `#R 承諾前フォロー` は承諾を待つ段階。`#R クローズお礼` は締めの段階。 | まだ承諾前か、締めか |

## 運用メモ
- 時刻指定がなければ、購入後フェーズの次回時刻は「JST現在時刻+2時間」を自動補完する。
- `#0$` / `#1$` / `#2$` / `#3$` は `#$0` / `#$1` / `#$2` / `#$3` の誤記エイリアスとして扱う。
- `#S` は「録画開始」、`#M` は「しおり」、`#C` は「録画終了」のイメージで使う。
- `#CL` は「直近の返信文レビュー依頼」のショートカットとして使う。
- `#GE` は「直近の返信文を Gemini で見る」ショートカットとして使う。
- `#M` を打つ目安:
  - 同一原因 / 別原因の判定が出た時
  - 5,000円 / 15,000円の判断が固まった時
  - 感情注意が必要だと分かった時
  - 納品方針が固まった時
- `#C` だけでも最終要約は作れるが、途中の判断根拠を残したい案件では `#M` を使う。
- `#R 進捗返信` は残しているが、今後の途中報告は `#P` を正本として使う。
- 保存ルールの詳細は `/home/hr-hm/Project/work/ops/common/case-record-template.md` を参照する。
- 見積り返信や感情注意の練習・検証は `/home/hr-hm/Project/work/ops/tests/README.ja.md` を起点にし、自然文ケースは `prequote-test-cases.txt` を優先して使う。
