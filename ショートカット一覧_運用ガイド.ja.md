# ショートカット一覧（運用早見表）

## まずはこれだけ（最小運用セット）
- `#$0`
- `#R 受領返信`
- `#R 進捗返信`
- `#R 方向性返信`
- `#R 正式納品案内`
- `#R クローズお礼`

## 一覧表
| タグ | フェーズ | 使うタイミング | 返す内容（要約） | メモ |
|---|---|---|---|---|
| `#$0` | 購入直後 | 直接購入でトークルームが開いた直後 | 初回あいさつ + 必須3点ヒアリング + 見立て目安 | 受注開始の起点 |
| `#$1` | 見積り提案 | 合意後、提案ボタン送信前 | 提案を送る予告の一言 | 提案ボタンの直前 |
| `#$2` | 見積り提案 | 提案ボタン送信後 | 購入導線の案内 | 「提案を購入する」へ誘導 |
| `#$3` | 見積り提案 | 提案後に未返信 | 急かさないフォロー | 見積り段階専用 |
| `#R 受領返信` | 調査 | 資料受領直後 | 受領確認 + 確認開始の一言 | 放置不安を防ぐ |
| `#R 進捗返信` | 調査 | 受領後30〜60分 | 途中報告 + いま見ている点 + 次回時刻 | 無音を避ける |
| `#R 方向性返信` | 調査 | 原因候補が見えた時 | 最初の見立て + 次アクション + 次回時刻 | 技術説明の芯 |
| `#R 納品前確認` | 納品前 | 正式納品ボタンを押す前 | 修正点の要約 + 最終確認依頼 | 無言納品を防ぐ |
| `#R 正式納品案内` | 納品後 | 正式納品ボタンを押した直後 | 承諾/差し戻し案内 + 約3日未操作で自動承諾 | クローズ条件を明示 |
| `#R 承諾前フォロー` | 納品後 | 納品後に返信が止まった時 | 操作案内つきの軽い確認 | 急かさない文面 |
| `#R 差し戻し返信` | 再対応 | 差し戻し受領直後 | 受領 + 修正方針 + 次回時刻 | 不満の火消しを優先 |
| `#R クローズお礼` | クローズ | 承諾前後の締め | お礼 + 任意の評価お願い（1回のみ） | 最後の印象作り |

## 使い分け解説表（迷いやすい組み合わせ）
| 迷う組み合わせ | 使い分け基準 | 判断ポイント |
|---|---|---|
| `#$0` / `#R 受領返信` | `#$0` は購入直後の初回。`#R 受領返信` は資料が届いた直後。 | 初回開始か、資料受領後か |
| `#R 進捗返信` / `#R 方向性返信` | `#R 進捗返信` は調査中報告。`#R 方向性返信` は原因候補を示せる段階。 | 見立てが出せるか |
| `#R 納品前確認` / `#R 正式納品案内` | `#R 納品前確認` はボタン前。`#R 正式納品案内` はボタン後。 | 納品ボタンを押したか |
| `#$3` / `#R 承諾前フォロー` | `#$3` は見積り提案後の未返信。`#R 承諾前フォロー` は正式納品後の未返信。 | 購入前か納品後か |
| `#R 承諾前フォロー` / `#R クローズお礼` | `#R 承諾前フォロー` は承諾を待つ段階。`#R クローズお礼` は締めの段階。 | まだ承諾前か、締めか |

## 運用メモ
- 時刻指定がなければ、購入後フェーズの次回時刻は「JST現在時刻+2時間」を自動補完する。
- `#0$` / `#1$` / `#2$` / `#3$` は `#$0` / `#$1` / `#$2` / `#$3` の誤記エイリアスとして扱う。
