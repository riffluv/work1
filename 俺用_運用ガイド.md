# 俺用_運用ガイド

これは、自分が実際にやる操作だけに絞った短縮版です。  
細かい内部処理は覚えなくて大丈夫です。

## 1. まず覚えるのはこれだけ

### よく使うタグ
- `#R` : 送る返信文を作る
- `#A` : 分析だけ出す
- `#S` : 購入後に案件を開始する
- `#M` : 重要判断を保存する
- `#P` : 購入後の進捗返信を作る
- `#C` : クローズする
- `#CL` : Claude に監査させる
- `#GE` : Gemini に監査させる

### `#A` と `#R` の違い
- `#A` : 方針を見る。「どう返すべきか」「何を先に言うか」を見る
- `#R` : 完成文を見る。そのまま送れる文面を作る

## 2. 基本の流れ

### 見積もり相談が来た時
自分がやること:
1. 相手文を貼る
2. 返信文がほしければ `#R`
3. 分析だけ見たければ `#A`

例:
```text
#R
Stripe のWebhookが本番だけ失敗します。料金も知りたいです。
```

分析だけなら:
```text
#A
この相談、まずどう返すべきか見て
```

### 返信にひとこと条件を足したい時
`#R` の下にそのまま書いてよいです。

例:
```text
#R
ここは柔らかめにして
```

```text
#R
質問は1つだけにして
```

## 3. 購入された時
自分がやること:
1. `#S` を使う
2. 必要ならそのまま `#R` も一緒に使う

例:
```text
#S #R
```

意味:
- 案件記録を開始する
- active case が決まる
- mode が実装用に切り替わる
- 最初の返信も作れる

## 4. 実装中
自分がやること:
1. コードやログを渡す
2. 重要判断だけ `#M`
3. 進捗返信が必要なら `#P`

### 進捗返信
```text
#P
```

### 重要判断を残す
```text
#M
decision: 同一原因として継続
reason: Webhook と受信処理の差だけで説明できる
next_action: route.ts を確認
```

## 5. 納品前後
自分がやること:
1. 納品前確認が必要なら `#R 納品前確認`
2. 正式納品後の案内なら `#R 正式納品案内`
3. 差し戻しが来たら `#R 差し戻し返信`

例:
```text
#R 納品前確認
```

```text
#R 正式納品案内
```

## 6. クローズ時
自分がやること:
1. `#C`

例:
```text
#C
```

意味:
- 案件が closed に移る
- mode が `coconala` に戻る

## 7. 案件切替と memory

ここはかなり重要です。  
見積もり相談だけならあまり気にしなくて大丈夫ですが、購入後の実案件を2件以上回すなら必須です。

### 何のためにあるか
- 前回どんな約束をしたか
- 前に何時までと伝えたか
- 何回目の follow-up か

を案件ごとに分けるためです。

### memory の保存先
- 案件ごとの正本  
  `/home/hr-hm/Project/work/ops/cases/open/{case_id}/reply-memory.json`
- 直近の fallback  
  `/home/hr-hm/Project/work/runtime/replies/latest-memory.json`

### memory に入るもの
- `previous_assistant_commitment`
- `previous_deadline_promised`
- `followup_count`
- `prior_tone`
- `commitment_fulfilled`

### いつ使うか
- 購入後の follow-up 系だけ
- 見積もり相談では基本使わない

### 何をすればいいか
- 見積もり相談だけなら、ほぼ何もしなくて大丈夫
- 購入後案件が1件だけなら、そのままでも回せる
- 購入後案件が2件以上あるなら、返す前に必ず active case を切り替える

### 切り替え方
正本はこれです。

```bash
/home/hr-hm/Project/work/scripts/case-switch.sh 1
```

または case_id を直接指定します。

```bash
/home/hr-hm/Project/work/scripts/case-switch.sh 20260407-01-talkroom
```

起動プロンプト上では `cs 001` の短縮表記が出てきますが、正本として覚えるのは  
`/home/hr-hm/Project/work/scripts/case-switch.sh` だけで十分です。

### ここを切り替えないとどうなるか
- 前の案件で約束した時刻が混ざる
- `followup_count` が別案件のまま残る
- 前の案件のトーンや文脈で返す可能性がある

なので、購入後案件を並行で返すなら `case-switch.sh` は実質必須です。

### 今どこを見てるか確認したい時
切り替える前に、
- 今どの案件が active か
- 今の mode は何か
- follow-up 何回目か
- 次アクションは何か

を一発で見たい時は、これを使います。

```bash
/home/hr-hm/Project/work/今どこ.sh
```

これは日本語名の確認コマンドです。中では  
`/home/hr-hm/Project/work/scripts/internal-os-status.sh`  
を呼んでいます。

見る時の使い分け:
- 案件を切り替える → `case-switch.sh`
- 今どの案件を見ているか確認する → `今どこ.sh`

## 11. どうしても迷った時の最短手順
1. 相手に返す文がほしいなら `#R`
2. まず整理だけしたいなら `#A`
3. 購入後なら `#S`
4. 実装中の途中報告なら `#P`
5. 終わりなら `#C`
6. 購入後の別案件に返すなら `case-switch.sh`

これで大体回せます。
