## 4. 現在までの重要レンズ

### transaction_model_gap

日本語のツギハギ感を、単なる文体ではなく、取引構造の欠落として見るレンズです。

見るもの:

- 支払い状態
- 作業開始条件
- 見積り提案
- 購入
- 成果物返却
- closed 後の再相談導線
- 無料対応期待
- 追加料金の自動発生に見えないか

これはかなり重要です。

ただし writer rule として常駐させすぎず、再発する deterministic 事故だけ validator / gold へ戻してください。

### buyer_state_ack_gap

症状だけを受けて、buyer の状態シグナルを落としていないかを見るレンズです。

対象:

- 疲弊
- 不信
- 再発不安
- 費用不安
- 進捗不安
- 過去の他者修正失敗

やること:

- 1文だけ、事実ベースで受ける
- 過剰謝罪しない
- 責任認定しない
- 無料対応を約束しない

例:

```text
同じような症状がまた出ていて、前回分との関係や費用面も気になる状況かと思います。
```

### response_weight_mismatch

返信量が、buyer の文量・質問の重さ・求めている回答量と合っているかを見るレンズです。

注意:

- safety atom を削らない
- 返金 / closed / scope / phase は短くしすぎない
- yes/no や対象可否だけなら短く返す
- buyer が「短くでいい」と言っているなら、実際に短くする

batch-36 では、この観点により初の全7件 r0 通過が出ました。

### bot感 / パーツ貼り付け感

独立レンズとして増やしすぎず、次の既存観点に紐づけて扱ってください。

- response_weight_mismatch
- buyer_state_ack_gap
- route_state / route_match_decided
- Japanese naturalness

注意する表現:

- `はい、できます` が文脈途中で浮く
- `合っています` が診断発表のように見える
- 共感文の直後に価格文が貼り付いたように見える
- buyer が正しい route を選んでいるのに、こちらが「その方が合っています」と診断する

## 5. `#BR` で守る route 原則

### bugfix-first

今止まっている不具合が主目的なら `bugfix-first`。

例:

- 注文履歴が作られない
- Webhook が失敗している
- 支払い後にDB更新されない
- まず止めたい、復旧したい

重要:

- `コードが読めない` だけで handoff に逃がさない
- 15,000円の不具合修正内では、修正ファイル・変更箇所・確認順の軽い説明は可
- 本格的な引き継ぎメモは別
- `原因だけ安く見てメモだけ` という安い diagnosis-only 入口を作らない

### handoff-first

今壊れておらず、次の担当者へ渡す・危険箇所を把握する・主要1フローを整理したいなら `handoff-first`。

例:

- 決済から注文作成までを整理したい
- 次の外注先へ渡すメモがほしい
- 触ると危ないファイルを知りたい
- Webhook secret / APIキー / env vars の使われている場所を整理したい

重要:

- 25,000円は主要1フローまで
- リファクタリング、ファイル削除、関数名変更、コード修正は含めない
- secret / APIキーの値そのものは扱わない。キー名・場所・用途まで
- 複数 repo でも直接つながる1フローなら候補にできるが、全体監査へ広げない

### neither

どちらにも入らないもの。

例:

- Stripe Connect 導入
- Customer Portal 導入
- Stripe Tax 追加
- CSVボタン追加
- 自動リトライ機能
- 顧客対応FAQ
- スタッフ操作マニュアル
- 正式仕様書
- 社内運用マニュアル一式

重要:

- bugfix / handoff に吸収しない
- 「メモも作ってほしい」と言われても、新機能に付随する説明は neither

### bridge / mixed

bugfix と handoff が混ざる場合は、主目的で順番を決めます。

- active defect があるなら bugfix-first
- active defect がないなら handoff-first
- bugfix 後に必要なら handoff は別
- handoff 中にバグが見つかっても、その場で修正までは含めない
- bundle / 値引き / 一括価格は約束しない
