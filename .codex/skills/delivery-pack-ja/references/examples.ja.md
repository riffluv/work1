# 出力例（短縮）

## 例: 標準納品（修正済みZIP）
診断レポート
- 結論: ご依頼の問題は解消しました
- 原因: Webhook署名検証のシークレット参照先が誤っていた
- 修正内容: `api/webhook.ts` で環境変数参照と例外処理を修正
- ご確認の手順: 決済実行 -> webhook受信 -> DB反映確認

納品メッセージ
このたびはお待たせしました。ご依頼の問題について、今回の修正内容をお送りします。
- 今回の問題と原因: Webhook署名検証の参照先誤りで、決済後の反映処理が止まっていました
- 修正した内容: 参照先と例外処理を修正しました
- お送りするもの: 修正済みZIP + 診断レポート
- ご確認の手順: 決済後に反映ログを確認してください
問題なければ「承諾」をお願いします。修正が必要な場合は「差し戻し」でご連絡ください。
納品から約3日間操作がない場合は、自動的に承諾となりトークルームが閉じます。

## 例: ZIP + patch併用（Git運用者向け）
診断レポート
- 結論: 本番で発生していたAPI連携エラーは解消しました
- 原因: 本番のみ API base URL が空になっていた
- 修正内容: `config.ts` の環境判定とfallbackを修正
- 今回の修正が関係する部分:
  - 修正の対象: 外部API連携フローの設定読み込み
  - 今回は触れていない部分: 外部API 側の契約内容や管理画面設定
  - 今回の修正は、アプリ側で本番設定を正しく読む部分を対象にしています

納品メッセージ
このたびはお待たせしました。本番で発生していたAPI連携エラーについて、修正内容をお送りします。
- 今回の問題と原因: 本番だけ設定値が空になっていました
- 修正した内容: 環境判定とfallbackを修正しました
- お送りするもの: 修正済みZIP + patch + 診断レポート
- 適用方法: 通常は修正済みファイルを差し替えてください。patchはGit運用者向け補助です。
- ご確認の手順: 本番相当環境でAPI連携を再実行してください
問題なければ「承諾」をお願いします。修正が必要な場合は「差し戻し」でご連絡ください。
納品から約3日間操作がない場合は、自動的に承諾となりトークルームが閉じます。

## 例: 非エンジニア向け（ZIPのみ）
診断レポート
- 結論: 決済後に反映されない問題は解消しました
- 原因: 決済後の処理で、設定値の参照先が間違っていました
- 修正内容: 設定値の参照先を正しい場所に修正しました（1ファイル）
- 今回の修正が関係する部分:
  - 修正の対象: 決済完了後の反映処理
  - 今回の修正は、支払い後に会員情報へ反映する部分を対象にしています
- ご確認の手順: テスト決済を1件実行 -> 管理画面で反映を確認

納品メッセージ
このたびはお待たせしました。ご依頼の問題について、今回の修正内容をお送りします。
- 今回の問題と原因: 決済後の設定値参照先が間違っていました
- 修正した内容: 正しい参照先へ修正しました
- お送りするもの: 修正ファイル + 診断レポート
- ご確認の手順: テスト決済を実行し、管理画面で反映を確認してください
問題なければ「承諾」をお願いします。修正が必要な場合は「差し戻し」でご連絡ください。
