# 出力例（短縮）

## 例: 標準納品（修正済みZIP）
診断レポート
- 原因: Webhook署名検証のシークレット参照先が誤っていた
- 修正内容: `api/webhook.ts` で環境変数参照と例外処理を修正
- 影響範囲: 決済成功後の反映処理
- 確認手順: 決済実行 -> webhook受信 -> DB反映確認

納品メッセージ
正式な納品をお送りします。
- 納品物: 修正済みZIP + 診断レポート + 検証メモ
- 確認手順: 決済後に反映ログを確認
問題なければ「承諾」をお願いします。修正が必要な場合は「差し戻し」でご連絡ください。
納品から約3日間操作がない場合は、自動的に承諾となりトークルームが閉じます。

## 例: ZIP + patch併用（Git運用者向け）
診断レポート
- 原因: 本番のみ API base URL が空になっていた
- 修正内容: `config.ts` の環境判定とfallbackを修正
- 影響範囲: 外部API連携フロー

納品メッセージ
正式な納品をお送りします。
- 納品物: 修正済みZIP + patch + 診断レポート + 検証メモ
- 適用方法: 通常は修正済みファイルを差し替えてください。patchはGit運用者向け補助です。
- 確認手順: 本番相当環境でAPI連携を再実行
問題なければ「承諾」をお願いします。修正が必要な場合は「差し戻し」でご連絡ください。
納品から約3日間操作がない場合は、自動的に承諾となりトークルームが閉じます。

## 例: 非エンジニア向け（ZIPのみ）
診断レポート
- 原因: 決済後の処理で、設定値の参照先が間違っていました
- 修正内容: 設定値の参照先を正しい場所に修正しました（1ファイル）
- 影響範囲: 決済完了後のデータ反映
- 確認手順: テスト決済を1件実行 -> 管理画面で反映を確認

納品メッセージ
正式な納品をお送りします。
- 納品物: 修正ファイル + 診断レポート + 検証メモ
- 確認手順: テスト決済を実行し、管理画面で反映を確認してください
問題なければ「承諾」をお願いします。修正が必要な場合は「差し戻し」でご連絡ください。
納品から約3日間操作がない場合は、自動的に承諾となりトークルームが閉じます。
