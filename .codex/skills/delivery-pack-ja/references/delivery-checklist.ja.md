# 納品前チェックリスト

## 必須チェック
- [ ] 原因が1〜3行で説明されている
- [ ] 修正内容が対象単位で書かれている
- [ ] 確認手順が依頼者環境向けに書かれている
- [ ] 納品形式（標準: 修正済みZIP / 任意: patch）が合意済み
- [ ] patchを併記する場合、優先適用方法（通常はZIP差し替え）を明記した
- [ ] 正式納品メッセージに差し戻し案内がある
- [ ] 正式納品メッセージに「約3日間未操作で自動承諾クローズ」を明記した
- [ ] 提供途中・未提供の状態で正式納品を送っていない
- [ ] 評価依頼を入れる場合、1回のみで任意トーンになっている

## セキュリティチェック
- [ ] `.env` 値を含めていない
- [ ] APIキー/Webhook秘密値を含めていない
- [ ] 顧客データや内部URLを含めていない

## 破綻防止チェック
- [ ] 対応内容と納品物を混ぜて書いていない
- [ ] 依頼者の次アクションが1行で明記されている
- [ ] 追加対応がある場合、見積り更新導線が明記されている
