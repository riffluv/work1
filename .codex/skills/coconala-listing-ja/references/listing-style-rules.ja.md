# 出品文スタイルルール（日本語）

## 基本トーン
- 標準は「丁寧で親しみやすい」。
- 過剰敬語・過剰カジュアルを避ける。
- 抽象語より具体語（対象、範囲、価格、納期）を使う。
- 読み手の不安を下げる一文を入れる（例: ご相談だけでも大丈夫です）。

## 人間向け明瞭化（最優先）
- 1文目で「何をいくらで提供するか」を言い切る。
- 1文1意味にする（対応内容・納品物・条件を1文に混在させない）。
- 「原因切り分けから修正差分、検証手順の共有まで」のような長い名詞列は2文に分割する。
- 重要な線引き（基本料金内 / 追加対応）を見出し直下で明示する。
- 初見の購入者が3秒で理解できる長さを優先する。

## 推奨構成（プロフィール）
1. ひとことアピール（1行）
2. 自己紹介（何が得意か）
3. 対応内容（箇条書き）
4. 技術スタック（箇条書き）
5. 実績（箇条書き）
6. 進め方（3〜5行）
7. 対応範囲外（箇条書き）
8. 締めの一文（相談導線）

## 推奨構成（出品サービス本文）
1. 何をいくらで提供するか（冒頭2行）
2. こんな症状に対応（箇条書き）
3. 対応内容（箇条書き）
4. 不具合1件の定義
5. 納品物
6. 標準範囲外
7. 最終確認フロー
8. 有料オプション
9. 購入時のお願い
10. FAQ（Q/A）

## FAQルール
- 原則 `Q:` / `A:` 形式。
- 1回答は短く、結論から書く。
- よく聞かれる順に並べる（価格/範囲/追加料金/秘密情報/デプロイ）。

## NG表現と言い換え（抜粋）
- `初心者ですが頑張ります` -> `1件ずつ丁寧に対応します`
- `何でもやります` -> `対応範囲をご相談のうえ決定します`
- `絶対に直します` -> `原因特定と修正案提示まで責任を持って対応します`
- `とても重要です` -> `重要な理由はAとBです`
- `丁寧に対応します` -> `初回返信は24時間以内（通常数時間以内）です`
- `できません` -> `現状の範囲外です。追加対応なら可能です`
- `無理です` -> `現条件では難しいため、代替案を提案します`
- `ご査収ください` -> `添付をご確認ください`
- `幸甚に存じます` -> `助かります`
- `差し支えなければご教示賜り` -> `可能なら教えてください`
- `〜させていただきます`（多用） -> `〜します` / `〜いたします`
- `当方` -> `私`（または主語省略）
- `お客様`（個別文） -> `{名前}様`
- `あなた` -> `{名前}様`（または主語省略）

## 採用しないルール
- 絵文字・顔文字の多用
- 感嘆符の連発
- 断定的な誇張（No.1、100%など）
- 事実根拠のない実績表現
- 固定条件にない新料金（例: 調査費2,000円）や独自オプションの追記（外部レポートの提案値も、固定条件に追加されるまでは採用しない）
- 外部共有を前提にした案内（GitHub/Drive/外部URL）
- 見積り相談経由で有料オプション追加を案内する文面

## 仕上げチェック
1. 固定条件（価格・範囲・禁止事項）を保持したか
2. 読み手が最初の3行で提供内容を理解できるか
3. 具体語（価格、範囲、納品物、返信目安）が入っているか
4. 過剰敬語とAI臭ワードが残っていないか
5. 次アクション（相談、見積り、購入）が明確か
6. 1文目が「何をしてくれるか」を直接答えているか
7. 1文に情報を詰め込み過ぎていないか（必要なら2文へ分割）
