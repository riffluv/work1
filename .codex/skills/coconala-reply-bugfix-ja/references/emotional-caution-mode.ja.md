# 感情注意モード

## 発動条件
- `coconala-intake-router-ja` の出力に `emotional_caution: true` がある場合
- または、購入者文に明らかな不満・苦情の語調がある場合

## 通常モードとの違い
- 先頭: 結論や事実整理より先に、不便の認知を置く
- 質問数: 最大3点ではなく、最初は1点に絞る
- 質問形式: 箇条書きより地の文を優先する
- 料金 / 規約 / 境界説明: 第2通以降に回せるなら回す
- 次アクションと時刻目安: 省略しない

## 生成ルール
1. 先頭は短い不便の認知または状況確認で始める。closed 後の前回ミス疑いなど、原因未確認で責任認定に見えやすい場面では `ご不便をおかけしています` より `ご不便な状況かと思います` `まずは状況を確認します` を優先する
2. 相手の言葉を1つ拾ってから進める（例: `また同じ症状が出たとのこと`）
3. 受け止めで状況を長く復唱しすぎない。closed 後・返金・苦情寄りの場面では、`〜ですよね` 型の感情代弁より、`ご不便な状況かと思います` `まずは状況を確認します` `いま確認している点を短く共有します` のような状況認知と現在地を優先する
4. 受け止め直後に `大丈夫です` を置かない。安心は `まずはここを見ます` `この順で進めます` のように具体策で示す
5. 最初の依頼は1点だけにし、できるだけ箇条書きを避ける
6. `お気持ちももっともです` のような感情の要約承認は避ける。事実の受領と不便の認知で止める
7. `責めたいわけではない` `文句ではない` `苦情ではない` のような否定されたネガティブ意図は、そのまま復唱しない。`前回の修正との関係確認` `似たエラーの状況確認` など、実務上の確認目的へ要約する
8. `今日中に無料で` `返金してほしい` などの強い要求語はそのまま復唱せず、`作業可否` `費用や返金の扱い` `前回修正との関係` へ要約する
9. `前回の対応の続きとして確認する` ニュアンスで書く
10. この返信では、料金・見積り・保証・規約説明を前面に出しすぎない。ただし、無料/返金/今日中対応を聞かれている場合は、論点を消さず未断定として返す
11. `本日[時刻]までに見立てをお返しします` など、次の見通しは必ず入れる

## 変えないこと
- 固定条件（価格、スコープ、範囲外、セキュリティ）は変えない
- 事実を曲げない
- 非を認定する表現は使わない
- 無償対応を約束しない

## 禁止表現
- `保証はしていません` を最初の返信で使わない
- `新規ご依頼をお願いします` を最初の返信で使わない
- `前回は承諾いただいています` を使わない
- `規約上は` で始めない
- `ご了承ください` で締めない
