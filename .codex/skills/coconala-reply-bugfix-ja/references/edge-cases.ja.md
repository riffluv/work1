# 金銭・状態遷移のエッジケース

## 値引き・確認範囲・追加支払い
- 値引き要求が来た場合:
  - 基本料金15,000円は崩さない
  - 公開 bugfix に確認専用の別料金案内は作らない
  - `今回だけ` `特別に` などの文面を作らない
- 見積り経路で追加が発生した場合:
  - 有料オプション追加前提で書かない
  - `進め方と費用を先に相談する / 別件として分ける` の2択で案内する
- 判定前に確認が必要なとき:
  - 15,000円は調査だけで終わる形ではなく、原因確認から修正・確認手順・修正済みファイル返却まで進める前提で返す
  - `まず状況を見て、15,000円で進められるかを返す` と案内する
  - 最初から修正希望が強い相談を、未公開の別料金へ倒さない

## キャンセル・quote_sent
- キャンセル寄りの相談:
  - まず理由を1点だけ確認する
  - 進め方変更で吸収できる余地があるか見る
  - それでもキャンセル希望なら、ココナラの手続きに沿って進める旨だけ簡潔に伝える
- `quote_sent` 状態:
  - 内容変更、期限切れ、金額質問は急かさず整理する
  - 再提案が必要ならその前提を短く伝える

## `closed` 後の再相談
- 既存トークルームの再開前提で書かない
- `新しい相談` `新規依頼` を最初の1通で前面に出さない
- まず `トークルームが閉じているため、ここから改めて確認する` と伝える
- ただし、前回未解決感が強い文面では `規約上の境界` を先に出しすぎない
- 先に `再発して困っていること` と `前回確認時は動いていたこと` を受け止める
- 最初の依頼は1点だけに近づける（スクショ1枚 + 同じ症状かどうか）
- 最初の1通で料金や保証の話を先出ししない
- ただし、相手が追加料金の有無を直接聞いている場合は、その質問を無視しない
- この場合は `追加の費用がかかるかどうかは、まず状況を確認させてください。前回の修正との関係を見たうえで、進め方を整理します。` のように、方向性だけ答えて断定は避ける
- `はい、かかります` `いいえ、無料で見ます` の即答は避ける
- 確認前にこちらの過失や無料対応を認める表現は使わない
- 同一原因の再発に見える場合でも、公開 bugfix の案内は 15,000円 / 保留 / 断る の枠内で返す
