対象: `RE-2026-04-27-bugfix-21-normal-with-phase-sentries-r2`

目的:

- 普通寄りの見積り相談で、直近の closed / transaction_model_gap 反映後も文面が硬くなっていないかを見る
- prequote の基本型で、価格・範囲・48時間目安・一息回答が自然に出るか確認する
- quote_sent / purchased / delivered / closed を1件ずつ混ぜ、phase contract と transaction_model_gap の接続漏れが残っていないか見る
- `surface_overexposure` の反動で、必要な境界説明が削れすぎていないか確認する
- `handoff-25000` や 25,000円側の外向け導線が出ていないか確認する

今回の構成:

- B01〜B03: 普通寄りの prequote
- B04: quote_sent の入金前作業境界 sentry
- B05: purchased の進捗不安 sentry
- B06: delivered の承諾前後 sentry

監査で見てほしい点:

- 主質問に最初に答えているか
- 普通の prequote が、過剰に契約説明・監査項目説明っぽくなっていないか
- `15,000円 / 不具合1件 / 追加料金は勝手に発生しない / 未完成納品へ押し切らない` の軸が崩れていないか
- quote_sent で、入金前にコード確認・原因調査へ入っていないか
- purchased で、進捗不安に対して現在地が見えるか。`確認中です` の同義反復で終わっていないか
- delivered で、承諾前 / 承諾後 / closed 後の導線が混ざっていないか
- closed で、メッセージ上の確認材料と、コード修正などの実作業が分かれているか
- closed で、無料対応確約にも通常料金決め打ちにも寄りすぎていないか
- 文面がぶつ切り、丸引用、テンプレ反復、内部分類語になっていないか

戻し先の判定方針:

- `validator`: deterministic に再発し、機械的に落とせる事故
- `gold`: 判断順序として残す価値があり、テンプレ丸写しではなく参照 anchor にするもの
- `reviewer_prompt`: 常時本文に出すと長文化するが、監査レンズとして有効なもの
- `case_fix`: その case だけの文体・温度・言い換え
- `preference`: 好み差で、今回の batch 採用を止めるほどではないもの

出力形式:

1. まず一文で結論
   - `採用可 / 条件付き採用 / 要修正` のどれかを最初に明記してください
   - そのうえで、「今回はそのまま使ってよいか / どこを直してから使うべきか」を1〜2文で書いてください
2. 共通して良い点
   - 2〜4点まで
3. 危ない点 / 共通 drift
   - 本当に再発しそうなものだけ
   - 細かい好みより、実務リスクと buyer の違和感を優先してください
4. ケース単位の必須修正
   - 必須修正がなければ `必須修正なし` と明記してください
   - 必須 / 軽微 を分けてください
5. 学習判定
   - 必須 / 軽微 の指摘ごとに、必要な場合だけ短く添えてください
   - `直す単位`: `case_fix / pattern_candidate / rule_return_candidate / preference / reject`
   - `再発根拠`: `なし / 同batch内 / 過去batch同型 / 既存rule未反映疑い`
   - `既存rule hit`: `あり / なし / 不明`
   - `戻し先候補`: `batch / reviewer_prompt / gold / rule / service_facts / validator / none`
6. rule に戻すべき再発パターン
   - `reply-only / common` を分けてください
   - 再発性が弱いものは無理に rule 化しないでください
7. 採点
   - 必要なら `直答 / scope順守 / 情報節約 / 自然さ / 次アクション明確さ` を短く評価してください
8. 最後に短くまとめ
   - `今回は止めてよい / 1〜2点だけ直す / まだ危ない` のどれかを、人間向けに短くまとめてください

まだ `rule / validator / gold / reviewer_prompt` には反映しないでください。監査結果を見て、再発性があるものだけ最小反映します。
