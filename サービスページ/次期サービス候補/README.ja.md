## 次の本命候補

### AI返信OS / 顧客対応運用設計
位置づけ:
- 今の返信システムを、他人のサービス運用へ移植する上位商品
- `skillを作る` のではなく、`AIを実務で事故らせない顧客対応構造を設計する` 商品
- テンプレ販売や軽いコンサルではなく、導入型の高単価商品として考える

本質:
- AIに返信を書かせること自体ではなく、
  - AIの思考を削らない
  - サービス内容を理解した状態で返す
  - ココナラ仕様を踏まえて事故らない
  - モグラたたきにならない
 という構造を作ることが価値

強み:
- ココナラの仕様を記憶領域として持たせられる
- サービス理解と価格・禁止事項を構造結合できる
- routing / scope / state / memory がある
- purchased 後の並列案件運用まで視野に入る
- 返信 lane と実装 lane を分けて、Codex の思考を殺さない

## 売り方の前提

やらない:
- 単なる `返信テンプレ集`
- 単なる `prompt 販売`
- 骨格そのものの安売り

やる:
- `あなたのサービスに合わせて、AIが事故らず返せる状態まで組み込みます`
- `ココナラ仕様を踏まえた顧客対応のAI運用を設計します`
- `顧客対応の属人化と炎上リスクを減らす返信OSを作ります`

外向け表現の候補:
- AI返信OS
- 顧客対応運用設計
- AI顧客対応導入
- 返信システム導入支援

※ 内部では `返信OS` と呼んでよいが、外向けでは `運用設計` や `導入支援` に寄せた方が伝わりやすい

## 価格感

### 本来価値

1サービス分の導入:
- `150,000〜300,000円`

複数サービス / memory / phase 管理込み:
- `300,000〜500,000円`

workspace レベル:
- `500,000〜1,000,000円+`

判断メモ:
- `3万〜8万` は安すぎる
- その価格だと、骨格価値に対して割が合わない
- 1サービス分をちゃんと入れる時点で、かなり骨格寄り
- 本当に売るなら、相手専用に組み込み、使える完成品として渡す方がよい

### ココナラ初出しの現実解

本体:
- `79,800円`
- 1サービス分
- ココナラ仕様の反映
- サービス理解
- 購入前 / 購入後の基本返信導線
- 購入後〜納品のフェーズ判定
- 範囲外・追加対応の基本判断
- 自然な日本語化

オプション:
- 文章トーン寄せ: `+25,000円`
- purchased memory / 継続文脈: `+45,000円`
- 追加サービス1件: `+65,000円`
- 文章学習 / 既存返信への寄せ: `+75,000円`
- case切替 / 複数案件運用: `+70,000円`

補足:
- ココナラの一覧で見える価格帯を踏まえると、初出しは `79,800円` が現実的
- これは `本来価値` より低めだが、初回の成約とレビュー獲得を優先した価格
- 最初の数件で評価が付いたら、段階的な値上げを前提にする

## 注意

- 薄い商品にすると価値が落ちる
- ただ冊子や方針メモを渡すだけでは弱い
- 相手が喜ぶのは、サービス理解が入り、実際に返せる状態になった時
- ただし、骨格をそのまま見せると分析・転用されやすい
- そのため、本丸の設計思想や深いレイヤーは隠し、利用面を完成品として渡す方向がよい
